<objective>
Create the FastAPI application skeleton that replaces Chainlit as the web server. This plan establishes the `web/` package with lifespan-managed graph singleton, session middleware, static file serving, Tailwind build pipeline, and Docker configuration changes.

Purpose: This is the foundation for all subsequent UI work. The lifespan singleton pattern prevents the critical per-request checkpointer pitfall identified in research. The session middleware enables per-conversation state without authentication.

Output: Running FastAPI app in Docker, serving static files, with graph/checkpointer initialized once at startup.
</objective>

  <name>Task 1: Create web/ package with FastAPI app, lifespan, session, and dependencies</name>
  <files>

  <action>
    1. Create `src/agentic_claims/web/__init__.py` (empty).

    2. Create `src/agentic_claims/web/main.py`:
       - Import `FastAPI`, `StaticFiles` from starlette, `SessionMiddleware` from starlette.middleware.sessions, `Jinja2Templates` from starlette.templating.
       - Define `lifespan` as an `@asynccontextmanager` async generator:
         - Call `getCompiledGraph()` from `agentic_claims.core.graph` — returns `(graph, checkpointerCtx)`.
         - Store `graph` on `app.state.graph`.
         - Store `checkpointerCtx` on `app.state.checkpointerCtx`.
         - Log "LangGraph graph and checkpointer initialized (lifespan singleton)" at INFO level.
         - `yield`.
         - In cleanup: `await checkpointerCtx.__aexit__(None, None, None)`.
         - Log "Checkpointer connection closed" at INFO level.
       - Create `app = FastAPI(title="Cognitive Atelier", lifespan=lifespan)`.
       - Add `SessionMiddleware` with `secret_key=getSettings().session_secret_key`, `session_cookie="agentic_session"`, `max_age=None` (session cookie), `same_site="lax"`, `https_only=False`.
       - Mount `StaticFiles(directory="static")` at path `/static`.
       - Create `templates = Jinja2Templates(directory="templates")`.
       - Store `templates` on module level so dependencies.py can access it, OR store on `app.state` in a startup event. Prefer storing as module-level since Jinja2Templates is sync and doesn't need lifespan.
       - Include `pagesRouter` from `agentic_claims.web.routers.pages`.

    3. Create `src/agentic_claims/web/session.py`:
       - Define `getSessionIds(request: Request) -> dict` as a function (not a FastAPI Depends — just a utility called by route handlers).
       - Check `request.session` for `thread_id` and `claim_id`.
       - If missing, generate new UUIDs via `uuid.uuid4()` and store in `request.session`.
       - Return `{"threadId": str, "claimId": str}`.

    4. Create `src/agentic_claims/web/dependencies.py`:
       - Define `getGraph(request: Request)` dependency that returns `request.app.state.graph`.
       - Define `getTemplates(request: Request)` dependency that returns the Jinja2Templates instance.

    5. Create `src/agentic_claims/web/routers/__init__.py` (empty).

    6. Create `src/agentic_claims/web/routers/pages.py`:
       - Create `router = APIRouter()`.
       - Define 4 GET routes that return `TemplateResponse`:
         - `GET /` -> `chat.html` with `{"request": request, "activePage": "chat", "session": getSessionIds(request)}`
         - `GET /dashboard` -> `dashboard.html` with `{"activePage": "dashboard", ...}`
         - `GET /audit` -> `audit.html` with `{"activePage": "audit", ...}`
         - `GET /review/{claimId}` -> `review.html` with `{"activePage": "review", "claimId": claimId, ...}`
       - Each route calls `getSessionIds(request)` to ensure session cookie is set.
       - Use `templates` from `main.py` module import (or from `request.app.state`).

    7. Add `session_secret_key` field to `Settings` in `src/agentic_claims/core/config.py`:
       - `session_secret_key: str = Field(..., description="Secret key for signing session cookies")`

    8. Add `SESSION_SECRET_KEY=change-me-to-a-random-secret` to `.env.example`.
       Also add it to `.env.local` (read current .env.local first, append if not present).

    IMPORTANT: Use CamelCase for all function names, variables, and parameters per project convention. Do NOT modify any existing code in core/, agents/, or infrastructure/ beyond adding the single `session_secret_key` field to Settings.
  </action>
  <verify>

    - `python -c "from agentic_claims.web.session import getSessionIds; print('OK')"` prints OK
    - `poetry run pytest tests/ -v` — all 61 existing tests pass
  </verify>
  <done>

       - Add to `[tool.poetry.group.dev.dependencies]`: `pytailwindcss = ">=0.3"`.
       - Keep `chainlit` dependency (it still exists alongside for now — Phase 7 removes it).
       - Run `poetry lock --no-update && poetry install` to update the lock file.

    2. Update `Dockerfile`:
       - Change CMD from `["chainlit", "run", "src/agentic_claims/app.py", "--host", "0.0.0.0", "--port", "8000"]` to `["uvicorn", "agentic_claims.web.main:app", "--host", "0.0.0.0", "--port", "8000"]`.
       - Add `COPY templates/ ./templates/` after the `COPY src/` line.
       - Add `COPY static/ ./static/` after the templates COPY.
       - Add `COPY tailwind.config.js ./` for the Tailwind config.
       - Remove `COPY .chainlit/ ./.chainlit/` and `COPY public/ ./public/` lines (Chainlit-specific, no longer needed for the app service — Chainlit can still run locally via `poetry run chainlit`).

    3. Update `docker-compose.yml` app service:
       - Add volume mounts for live reload of templates and static files:
         - `./templates:/app/templates`
         - `./static:/app/static`
       - These go alongside the existing `./src:/app/src` mount.
       - Update healthcheck test URL if needed (FastAPI serves at `/` which returns a template, so `curl -f http://localhost:8000/` should still work once templates exist).

    4. Create `tailwind.config.js` at repo root:
       - Use the exact Neon Nocturne design tokens from the Stitch HTML designs (all colors, fontFamily, borderRadius).
       - Set `darkMode: "class"`.
       - Set `content` to scan `["./templates/**/*.html"]` so Tailwind purges unused classes.
       - This is a standard Tailwind v3 config file (CommonJS `module.exports`), NOT the CDN-style `tailwind.config` object.

    5. Create `static/css/input.css`:
       - Add Tailwind directives: `@tailwind base;`, `@tailwind components;`, `@tailwind utilities;`.
       - Add the custom CSS from Stitch designs:
         - `.material-symbols-outlined` font-variation-settings
         - `.hide-scrollbar` webkit and firefox scrollbar hiding
         - `.intelligence-pulse` animation with `@keyframes pulse`
       - Add body-level utility layer if needed.

    6. Create `static/js/.gitkeep` as placeholder for vendored JS files.
       Download vendored JS files:
       - HTMX 2.0.8: `curl -o static/js/htmx.min.js https://unpkg.com/htmx.org@2.0.8/dist/htmx.min.js`
       - htmx-ext-sse 2.2.4: `curl -o static/js/sse.js https://unpkg.com/htmx-ext-sse@2.2.4/sse.js`
       - Alpine.js 3.14.9: `curl -o static/js/alpine.min.js https://unpkg.com/alpinejs@3.14.9/dist/cdn.min.js`

    7. Generate initial Tailwind CSS output:
       - Run `poetry run tailwindcss -i static/css/input.css -o static/css/output.css --minify`.
       - This creates the compiled CSS that will be served. The output.css will be mostly empty until templates with Tailwind classes exist (Plan 02).

    IMPORTANT: Do NOT remove chainlit from pyproject.toml — it coexists for now. The Dockerfile CMD change is the only thing that switches the app service from Chainlit to FastAPI.
