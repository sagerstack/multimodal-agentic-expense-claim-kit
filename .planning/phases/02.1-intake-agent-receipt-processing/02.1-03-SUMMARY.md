# Phase 2.1 Plan 03: Wire Intake Agent ReAct Loop Summary

**One-liner:** ReAct agent with ChatOpenAI wraps all 5 intake tools, Chainlit handles image uploads and interrupts, Evaluator Gate conditionally routes based on claim submission.

## What Was Built

- `src/agentic_claims/agents/intake/node.py` - `getIntakeAgent()` factory and `intakeNode()` wrapper
- `tests/test_intake_agent.py` - 4 tests for agent creation, tools, OpenRouter config, node interface

**Implementation:**

- `getIntakeAgent()`: Creates `ChatOpenAI` with OpenRouter config, collects 5 tools, returns `create_react_agent()` compiled graph

- Deprecated `OpenRouterClient` (added comment, kept for backward compatibility)

**ChatOpenAI Configuration:**
```python

    model=settings.openrouter_model_llm,
    base_url=settings.openrouter_base_url,
    api_key=settings.openrouter_api_key,
    temperature=0.7,
    max_retries=settings.openrouter_max_retries,
)
```

**Tools Available:**
1. `extractReceiptFields` - VLM extraction with quality gate
2. `searchPolicies` - RAG-based policy search
3. `convertCurrency` - MCP currency conversion
4. `submitClaim` - Dual-call database submission
5. `askHuman` - Human-in-loop clarification via `interrupt()`

### 2. Chainlit Image Upload and Interrupt Handling (Task 2a)
**File:** `src/agentic_claims/app.py`

**Changes:**
- `onChatStart()`: Added `claim_id` (uuid4) and `awaiting_clarification` (False) to session state
- `onMessage()`: Completely rewritten for image upload and interrupt/resume flow

**Image Upload Flow:**
1. Check `message.elements` for image attachments (mime starts with "image/")
2. Read bytes from `element.path` or `element.content`
3. Base64 encode and embed in HumanMessage content
4. Send "Receipt image received. Processing..." confirmation

**Interrupt/Resume Flow:**
1. If `awaiting_clarification=True`: Use `Command(resume={...})` to resume graph with user response
2. Else: Build HumanMessage and invoke graph normally
3. Check for `__interrupt__` in result
4. If interrupt present: Display question + data JSON, set `awaiting_clarification=True`
5. If no interrupt: Send AI messages from result

**Imports Added:**
- `base64` - Image encoding
- `json` - Interrupt payload formatting
- `Command` from `langgraph.types` - Resume support

### 3. Evaluator Gate and Graph Updates (Task 2b)
**Files:**
- `src/agentic_claims/core/graph.py` - Gate function and graph rewiring
- `tests/test_graph.py` - 6 tests (4 updated, 2 new)

**Evaluator Gate:**
```python
def evaluatorGate(state: ClaimState) -> str:
    """Route based on claimSubmitted flag."""
    if state.get("claimSubmitted", False):
        return "submitted"
    return "pending"
```

**Graph Topology Change:**
```
Before: START -> intake -> [compliance || fraud] -> advisor -> END

After:  START -> intake -> evaluatorGate -> {
            "submitted": postSubmission -> [compliance || fraud] -> advisor -> END
            "pending": END
        }
```

**Intermediate Node:**
- `postSubmission`: Pass-through lambda node (`lambda state: state`)
- Purpose: Enable fan-out to compliance+fraud after evaluator gate (LangGraph doesn't support list values in conditional edge routing dict)

## Key Architectural Decisions

### Decision 1: ChatOpenAI replaces OpenRouterClient
**Rationale:**
- `langchain_openai.ChatOpenAI` with `base_url` override provides equivalent functionality
- Better integration with LangChain ecosystem (tool binding, streaming, retries)
- Simplifies codebase (no custom retry logic needed)

**Trade-offs:**
- Kept `OpenRouterClient` for backward compatibility (marked deprecated)
- Will remove in future phase after confirming no dependencies

### Decision 2: Intermediate postSubmission node
**Rationale:**
- LangGraph conditional edges require scalar string values in routing dict
- Can't use `{"submitted": ["compliance", "fraud"]}` syntax
- Intermediate node enables standard fan-out edges

**Trade-offs:**
- Adds one extra node to graph (observable in stream output)
- Zero computational overhead (pass-through lambda)

### Decision 3: Base64 encoding in HumanMessage content
**Rationale:**
- Agent tools expect base64-encoded image strings
- Chainlit provides binary image data via `message.elements`
- Encoding at UI layer simplifies tool signatures

**Trade-offs:**
- HumanMessage content can be very long (base64 overhead)
- Could move to attachment mechanism in future if performance issues arise
