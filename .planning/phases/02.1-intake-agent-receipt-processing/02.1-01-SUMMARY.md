# Phase 02.1 Plan 01: VLM Receipt Extraction Foundation Summary

**One-liner:** VLM receipt extraction with OpenCV quality gate (blur/resolution), MCP client utility, and expanded ClaimState for intake conversation.

## What Was Built

Built the foundation layer for the Intake Agent: image quality detection, VLM extraction tool, MCP client utility, and expanded state schema.

**Image Quality Gate:**
- Laplacian variance blur detection (configurable threshold: 150.0)
- Resolution check (minimum: 800x600)
- Returns structured result with variance, resolution, acceptance flag, and reason

**VLM Extraction Tool:**
- LangChain tool decorated with @tool for ReAct agent compatibility
- Quality gate applied before VLM call (reject early)
- Multimodal message with base64 image + structured extraction prompt
- Returns fields (merchant, date, amount, currency, lineItems, tax, paymentMethod) + per-field confidence scores
- Error handling for invalid JSON responses

**MCP Client Utility:**
- Streamable HTTP transport connection to MCP servers
- ClientSession initialization and tool invocation
- Returns content list on success, error dict on failure (no exceptions)

**State and Config Expansion:**
- ClaimState: 4 new optional fields (extractedReceipt, violations, currencyConversion, claimSubmitted)
- Settings: 8 new fields (4 MCP URLs, 3 image quality params, 1 VLM confidence threshold)
- Environment files updated with localhost URLs for tests, Docker network URLs for local

**Dependencies Added:**
- langchain-openai (>=0.3) - ChatOpenAI for VLM
- opencv-python (>=4.0) - Image quality detection
- mcp (>=1.8) - MCP client SDK
- httpx (>=0.27) - Async HTTP (MCP dependency)
- numpy (>=1.26) - OpenCV dependency

## Decisions Made

**1. Quality Gate Before VLM**
- **Context:** VLM calls are expensive, blurry images produce poor results
- **Decision:** Check image quality (blur + resolution) before VLM call
- **Rationale:** Save API costs, provide clear feedback to user ("upload clearer image")
- **Impact:** Reduced unnecessary VLM calls, improved user experience

**2. Laplacian Variance for Blur Detection**
- **Context:** Multiple blur detection algorithms available
- **Decision:** OpenCV Laplacian variance with configurable threshold
- **Rationale:** Standard technique, fast, well-documented, tunable
- **Impact:** Simple implementation, effective detection

**3. Per-Field Confidence Scores**
- **Context:** VLM extraction quality varies per field
- **Decision:** Return confidence score for each extracted field
- **Rationale:** Enables selective human-in-loop (low-confidence fields only), no full rejection
- **Impact:** More granular quality control, better UX for partial extractions

**4. MCP Client Error Handling**
- **Context:** Connection failures should not crash agent
- **Decision:** Return error dict instead of raising exceptions
- **Rationale:** Agent can handle errors gracefully, retry, or escalate
- **Impact:** Robust error handling, no unhandled exceptions

## Architecture Insights

**Quality Gate Pattern:**
1. Fast cheap check (image quality) before expensive operation (VLM call)
2. Early rejection with actionable feedback
3. Reduces cost and latency

**MCP Client Design:**
- Single utility function for all MCP tool calls
- Consistent error handling across all servers
- Supports Streamable HTTP transport (required for FastMCP servers)

**State Schema Evolution:**
- Optional fields support incremental conversation building
- No defaults in TypedDict (fields can be missing)
- Agent functions populate fields as conversation progresses
