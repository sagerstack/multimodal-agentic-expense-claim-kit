### Gaps Summary

**1 critical gap blocks goal achievement:**

**Gap: claimSubmitted flag never set to True**

The intakeNode implementation has a logic flaw. Lines 79-81:

```python
if state.get("claimSubmitted", False):
    stateUpdate["claimSubmitted"] = True
```

This checks if claimSubmitted is ALREADY True in the input state, but it never will be. The submitClaim tool doesn't mutate state - it returns a dict to the agent. LangGraph tools don't have direct state mutation capability.

**Impact:**
- The evaluator gate (graph.py line 24) will always see claimSubmitted=False
- Graph will always route to END after intake, never to postSubmission→compliance+fraud
- Phase goal "submits a validated claim" is achieved (DB insertion happens), but "graph routes to compliance+fraud" never happens

**Missing implementation:**
The intakeNode needs to detect when submitClaim was called successfully. Options:
1. Parse result["messages"] to find ToolMessage with tool_call_id matching submitClaim and check if it has an error key
2. Inspect the ReAct agent's internal tool call results (if exposed)
3. Use a state schema that allows tools to update non-message fields (advanced LangGraph pattern)

**Why tests pass:**
Tests mock intakeNode to explicitly return `{"claimSubmitted": True}` (see test_graph.py lines 54-58), bypassing the real implementation's flaw.
