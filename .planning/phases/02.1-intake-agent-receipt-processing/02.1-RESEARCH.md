# Phase 2.1: Intake Agent + Receipt Processing - Research

**Researched:** 2026-03-25
**Domain:** LangGraph ReAct agents, VLM image extraction, Chainlit UI, MCP client integration
**Confidence:** HIGH

## Summary

Phase 2.1 builds the conversational Intake Agent that processes receipt images through VLM extraction, currency conversion, and policy validation — all within a Chainlit UI loop under 3 minutes. The agent uses LangGraph's ReAct pattern with human-in-the-loop interrupts for clarification and correction.

**Key architectural choices:**

1. **LangGraph ReAct agent via `create_react_agent`** (from `langgraph.prebuilt`, still supported in v1.0+) for the core conversational loop with domain tools (extractReceiptFields, searchPolicies, convertCurrency, submitClaim, askHuman).

2. **LangGraph `interrupt()` inside tools** enables human-in-the-loop clarification. Chainlit detects `__interrupt__` in graph results and prompts the user. User response resumes via `Command(resume=value)`.

3. **ChatOpenAI with OpenRouter base_url** replaces the existing custom OpenRouterClient. ChatOpenAI natively supports tool calling through OpenRouter's standardized interface. Configure with `base_url="https://openrouter.ai/api/v1"`.

4. **Separate VLM for extraction, text LLM for reasoning**: The ReAct agent uses the text LLM (OPENROUTER_MODEL_LLM) for tool calling. The `extractReceiptFields` tool internally calls the VLM (OPENROUTER_MODEL_VLM) with a structured extraction prompt optimized for receipt JSON.

5. **MCP servers via httpx Streamable HTTP**: Intake Agent tools call MCP servers at `http://{host}:{port}/mcp/` endpoints using `mcp.client.streamable_http.streamablehttp_client` from the MCP Python SDK.

6. **Laplacian variance for image quality check**: Reject blurry/low-resolution images before VLM extraction using OpenCV's variance of Laplacian method (threshold typically 100-1000, domain-dependent).

7. **ClaimState expansion**: Add fields for extracted receipt data, confirmation status, violations, and currency conversion results to support the intake conversation flow.

8. **Alembic migration for dual currency**: Add `originalCurrency` and `convertedAmount` columns to both `claims` and `receipts` tables to store foreign currency receipts with SGD conversions.

**Primary recommendation:** Use `create_react_agent` (not deprecated, still in `langgraph.prebuilt`) with ChatOpenAI configured for OpenRouter. Implement interrupt() in `askHuman` tool for clarification loops. Structure the VLM extraction prompt with JSON schema and field-by-field confidence scoring.

### Pattern 1: ReAct Agent as Node in Parent Graph

**What:** Use `create_react_agent` to build a subgraph, then add it as a node in the parent graph.

**When to use:** When the agent needs ReAct loop (think-act-observe) with tools, but is just one phase of a larger workflow.

**Example:**
```python
# Source: Context7 LangGraph documentation, adapted for Intake Agent
from langgraph.prebuilt import create_react_agent
from langchain_openai import ChatOpenAI
from langgraph.graph import StateGraph, END
from langgraph.checkpoint.postgres.aio import AsyncPostgresSaver

# Configure ChatOpenAI with OpenRouter
llm = ChatOpenAI(
    model="meta-llama/llama-3.1-8b-instruct:free",
    base_url="https://openrouter.ai/api/v1",
    api_key=settings.openrouter_api_key,
    temperature=0.7,
)

# Define tools
tools = [extractReceiptFields, searchPolicies, convertCurrency, submitClaim, askHuman]

# Create ReAct agent (returns a compiled graph)
intakeAgent = create_react_agent(
    model=llm,
    tools=tools,
    state_modifier="You are an expense claims intake assistant. Extract receipt fields, validate against policy, convert currency if needed, and confirm with the claimant before submitting.",
    checkpointer=checkpointer,  # Share checkpointer with parent graph
)

# Use agent as node in parent graph
def intakeNode(state: ClaimState) -> ClaimState:
    """Invoke intake ReAct agent and merge results into parent state."""
    # Pass relevant state to agent
    agentInput = {
        "messages": state["messages"],
    }

    # Invoke agent subgraph
    result = await intakeAgent.ainvoke(agentInput, config={"configurable": {"thread_id": state["claimId"]}})

    # Merge agent results back into parent state
    return {
        "messages": result["messages"],
        "extractedFields": state.get("extractedFields"),  # Updated by tools
        "status": "submitted" if state.get("claimSubmitted") else state["status"],
    }

# Wire into parent graph
builder = StateGraph(ClaimState)
builder.add_node("intake", intakeNode)
builder.add_edge("intake", "compliance")
```

**Key insight:** `create_react_agent` returns a compiled graph. Wrap it in a node function to integrate with parent state schema.

### Pattern 2: Tool with Interrupt for Human-in-the-Loop

**What:** Use `interrupt()` inside a tool to pause execution and request human input. Resume with `Command(resume=value)`.

**When to use:** When the agent needs user confirmation or correction (e.g., low-confidence VLM extractions, policy violations).

**Example:**
```python
# Source: Context7 LangGraph interrupts documentation
from langchain.tools import tool
from langgraph.types import interrupt, Command

@tool
def askHuman(question: str, extractedData: dict) -> dict:
    """Ask human to confirm or correct extracted data.

    Args:
        question: Clarification question for the user
        extractedData: Fields extracted by VLM with low confidence

    Returns:
        Confirmed or corrected data from human
    """
    # Pause execution and surface to Chainlit UI
    response = interrupt({
        "action": "clarify_fields",
        "question": question,
        "extracted_data": extractedData,
        "message": "Please confirm or correct these fields",
    })

    # When resumed, response contains user's corrections
    if response.get("action") == "confirm":
        return extractedData
    elif response.get("action") == "correct":
        return response.get("corrected_data", extractedData)
    else:
        return extractedData

# In Chainlit app.py
@cl.on_message
async def onMessage(message: cl.Message):
    graph = cl.user_session.get("graph")
    threadId = cl.user_session.get("thread_id")

    # Invoke graph
    result = await graph.ainvoke(
        initialState,
        config={"configurable": {"thread_id": threadId}}
    )

    # Check for interrupt
    if "__interrupt__" in result:
        interrupt_payload = result["__interrupt__"][0].value

        # Show clarification UI
        await cl.Message(content=interrupt_payload["message"]).send()
        await cl.Message(content=f"Extracted: {interrupt_payload['extracted_data']}").send()

        # User responds with corrections in next message
        # Store interrupt state to resume later
        cl.user_session.set("interrupt_state", result)
        return

    # If resuming from interrupt
    if cl.user_session.get("interrupt_state"):
        # User message contains corrections
        corrected = parseCorrectionFromMessage(message.content)

        # Resume graph with user's input
        result = await graph.ainvoke(
            Command(resume={"action": "correct", "corrected_data": corrected}),
            config={"configurable": {"thread_id": threadId}}
        )

        cl.user_session.set("interrupt_state", None)

    # Send agent messages
    for msg in result["messages"]:
        await cl.Message(content=msg.content).send()
```

**Gotcha:** LangGraph bug #4796 warns that interrupts in sub-graphs may restart from entry instead of resuming. Since we're using `create_react_agent` which returns a compiled graph, interrupts should work correctly within tools. The bug applies to manually created sub-graphs with nested interrupts.

### Pattern 3: VLM Structured Extraction with Confidence Scoring

**What:** Use a VLM with a structured prompt to extract receipt fields as JSON with per-field confidence scores.

**When to use:** Receipt processing, invoice extraction, any document where you need structured data from images.

**Example:**
```python
# Source: Roboflow blog post on VLM structured extraction, adapted
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage
import base64

async def extractReceiptFields(imageBytes: bytes) -> dict:
    """Extract structured receipt data from image using VLM.

    Returns:
        {
            "fields": {
                "merchant": "...",
                "date": "YYYY-MM-DD",
                "totalAmount": 123.45,
                "currency": "USD",
                "lineItems": [...],
                "tax": 12.34,
                "paymentMethod": "VISA"
            },
            "confidence": {
                "merchant": 0.95,
                "date": 0.87,
                "totalAmount": 0.99,
                ...
            }
        }
    """
    # Configure VLM (separate from reasoning LLM)
    vlm = ChatOpenAI(
        model=settings.openrouter_model_vlm,  # "meta-llama/llama-3.2-11b-vision-instruct:free"
        base_url=settings.openrouter_base_url,
        api_key=settings.openrouter_api_key,
        temperature=0.0,  # Deterministic extraction
    )

    # Encode image to base64
    imageB64 = base64.b64encode(imageBytes).decode('utf-8')

    # Structured extraction prompt
    prompt = """
Extract receipt information and return ONLY valid JSON in this exact structure:

{
  "fields": {
    "merchant": "exact merchant name as shown",
    "date": "YYYY-MM-DD format",
    "totalAmount": numeric_value_only,
    "currency": "3-letter code (USD, SGD, EUR, etc)",
    "lineItems": [
      {"description": "...", "quantity": 1, "unitPrice": 12.34, "amount": 12.34}
    ],
    "tax": numeric_tax_amount,
    "paymentMethod": "CASH or CARD or UNKNOWN"
  },
  "confidence": {
    "merchant": 0.0_to_1.0,
    "date": 0.0_to_1.0,
    "totalAmount": 0.0_to_1.0,
    "currency": 0.0_to_1.0,
    "lineItems": 0.0_to_1.0,
    "tax": 0.0_to_1.0,
    "paymentMethod": 0.0_to_1.0
  }
}

Extraction rules:
- Date must be YYYY-MM-DD. If only day/month visible, infer year from context.
- Amount fields are numbers without currency symbols.
- If field is unclear or not visible, use "unknown" for strings, 0.0 for numbers.
- Confidence score: 1.0 = perfectly clear, 0.5 = partially visible, 0.0 = not visible.
- Double-check: lineItems sum should approximately equal totalAmount - tax.

Return ONLY the JSON structure above, no additional text.
"""

    # Call VLM
    response = await vlm.ainvoke([
        HumanMessage(content=[
            {"type": "text", "text": prompt},
            {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{imageB64}"}}
        ])
    ])

    # Parse JSON response
    import json
    extracted = json.loads(response.content)

    return extracted
```

**Best practices:**
- Explicit JSON schema in prompt prevents malformed outputs
- Field-by-field confidence scoring enables smart clarification (only ask about low-confidence fields)
- Validation rules (date format, sum checks) reduce errors
- Temperature=0.0 for deterministic extraction
- Default values ("unknown", 0.0) prevent KeyError exceptions

### Pattern 4: MCP Client via Streamable HTTP

**What:** Use `mcp.client.streamable_http.streamablehttp_client` to call MCP servers over HTTP.

**When to use:** When MCP servers are deployed as separate Docker services with Streamable HTTP transport.

**Example:**
```python
# Source: MCP Python SDK documentation, Cloudflare blog post
from mcp.client.streamable_http import streamablehttp_client
from mcp import ClientSession
import httpx

async def callMcpTool(serverUrl: str, toolName: str, arguments: dict) -> dict:
    """Call an MCP tool via Streamable HTTP.

    Args:
        serverUrl: MCP server endpoint (e.g., "http://localhost:8001/mcp")
        toolName: Tool name from MCP server
        arguments: Tool arguments as dict

    Returns:
        Tool result from MCP server
    """
    # Custom httpx client with timeout
    httpClient = httpx.AsyncClient(timeout=30.0)

    # Connect to MCP server
    async with streamablehttp_client(serverUrl, http_client=httpClient) as (read_stream, write_stream, _):
        # Create MCP session
        async with ClientSession(read_stream, write_stream) as session:
            # Initialize connection
            await session.initialize()

            # Call tool
            result = await session.call_tool(toolName, arguments)

            return result.content

# Usage in Intake Agent tools
@tool
async def searchPolicies(query: str, limit: int = 5) -> list[dict]:
    """Search expense policies using RAG MCP server."""
    result = await callMcpTool(
        serverUrl="http://localhost:8001/mcp",  # RAG MCP server
        toolName="searchPolicies",
        arguments={"query": query, "limit": limit}
    )
    return result

@tool
async def convertCurrency(amount: float, fromCurrency: str, toCurrency: str = "SGD") -> dict:
    """Convert currency using Currency MCP server."""
    result = await callMcpTool(
        serverUrl="http://localhost:8002/mcp",  # Currency MCP server
        toolName="convertCurrency",
        arguments={"amount": amount, "fromCurrency": fromCurrency, "toCurrency": toCurrency}
    )
    return result
```

**Key insight:** FastMCP servers expose the MCP endpoint at `/mcp/` path. Clients connect using `streamablehttp_client` from the MCP SDK, which uses httpx under the hood.

### Pattern 5: Image Quality Check with Laplacian Variance

**What:** Detect blurry or low-resolution images using OpenCV's variance of Laplacian method before VLM extraction.

**When to use:** Pre-processing step to reject poor-quality images and prompt user to re-upload clearer images.

**Example:**
```python
# Source: Multiple OpenCV tutorials, adapted for receipt processing
import cv2
import numpy as np

def isImageBlurry(imageBytes: bytes, threshold: float = 100.0) -> tuple[bool, float]:
    """Detect if image is blurry using Laplacian variance method.

    Args:
        imageBytes: Raw image bytes (JPEG, PNG, etc)
        threshold: Variance threshold (lower = blurrier). Typical range: 100-1000

    Returns:
        (is_blurry, variance_score)
    """
    # Decode image
    nparr = np.frombuffer(imageBytes, np.uint8)
    image = cv2.imdecode(nparr, cv2.IMREAD_COLOR)

    # Convert to grayscale
    gray = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY)

    # Compute Laplacian variance
    laplacian = cv2.Laplacian(gray, cv2.CV_64F)
    variance = laplacian.var()

    # Compare against threshold
    isBlurry = variance < threshold

    return isBlurry, variance

# Usage in extractReceiptFields tool
@tool
async def extractReceiptFields(imageBytes: bytes) -> dict:
    """Extract receipt fields from image with quality check."""
    # Quality gate
    isBlurry, variance = isImageBlurry(imageBytes, threshold=150.0)

    if isBlurry:
        return {
            "error": "Image quality too low for extraction",
            "variance_score": variance,
            "recommendation": "Please upload a clearer image with better focus and lighting",
        }

    # Proceed with VLM extraction
    extracted = await vlmExtract(imageBytes)

    return extracted
```

**Threshold tuning:**
- Gather 20-30 sample receipts (sharp and blurry)
- Compute Laplacian variance for each
- Plot distribution and choose threshold between clusters
- Typical values: 100 (very conservative, rejects many), 500 (moderate), 1000 (lenient)
- Domain-dependent: receipt images from mobile phones may need lower threshold than scanned documents

### Anti-Patterns to Avoid

- **Using sub-graphs for HITL interrupts:** LangGraph bug #4796 warns interrupts in sub-graphs may restart instead of resume. Use interrupt() in tools within `create_react_agent` instead.

- **Hardcoding MCP server URLs:** All MCP endpoints should be in `.env.local` (RAG_MCP_URL, CURRENCY_MCP_URL, DB_MCP_URL, EMAIL_MCP_URL). Use Settings class to access.

- **Mixing VLM and LLM roles:** Don't use the VLM for conversational reasoning or the text LLM for image extraction. Separate models optimized for each task.

- **Low-confidence extractions without clarification:** If VLM confidence < 0.7 on critical fields (amount, date), always trigger `askHuman` tool to confirm. Don't submit claims with guessed data.

- **Ignoring image quality:** Sending blurry images to VLM wastes API calls and produces unreliable extractions. Always run Laplacian variance check first.

- **Using Chainlit AskActionMessage:** Known bugs (#2225, #2204) with AskActionMessage in callbacks. Use plain `cl.Message` + `on_message` handler for clarification instead.

## Don't Hand-Roll

Problems that look simple but have existing solutions:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| ReAct agent loop | Custom while loop with tool calling | `create_react_agent` from langgraph.prebuilt | Handles tool execution, error recovery, message formatting, state management. Battle-tested. |
| Human-in-the-loop pausing | Custom session state + polling | `interrupt()` from LangGraph | Native checkpointer integration, supports Command(resume=...), no polling needed. |
| Image blur detection | Custom edge detection heuristics | OpenCV Laplacian variance | Fast, reliable, well-studied method. Custom solutions miss edge cases. |
| MCP client calls | Raw HTTP POST/GET to MCP endpoints | `mcp.client.streamable_http` from MCP SDK | Handles bidirectional streaming, connection pooling, session management, error recovery. |
| Structured VLM extraction | Regex parsing of free-text VLM output | JSON schema in prompt + pydantic validation | VLMs trained on structured output. Schema in prompt ensures consistent format. |
| Currency conversion | Scraping exchange rate websites | Frankfurter API (already integrated) | Free, reliable, no API key, European Central Bank rates. |
| Database migrations | Manual ALTER TABLE scripts | Alembic | Tracks migration history, supports rollbacks, auto-generates migrations from ORM changes. |

**Key insight:** The Intake Agent has many moving parts (VLM, MCP, HITL, quality checks). Don't reinvent wheels — LangGraph, MCP SDK, and OpenCV provide production-ready primitives.

## Common Pitfalls

### Pitfall 1: Interrupt Doesn't Pause Execution

**What goes wrong:** Tool calls `interrupt()` but graph continues executing, ignoring the pause.

**Why it happens:** Checkpointer not configured, or interrupt() called outside a tool function.

**How to avoid:**
- Ensure `create_react_agent` has `checkpointer=checkpointer` argument
- Only call `interrupt()` inside `@tool` functions, not in node functions
- Use same checkpointer for parent graph and ReAct agent (shared state)

**Warning signs:**
- Graph completes instantly even when interrupt() is called
- `result["__interrupt__"]` is empty or missing

### Pitfall 2: Low VLM Confidence Ignored

**What goes wrong:** VLM extracts "merchant: unknown" with confidence 0.2, agent submits claim anyway.

**Why it happens:** No confidence threshold checks in tools, no clarification loop.

**How to avoid:**
- After VLM extraction, check confidence scores for critical fields (merchant, date, amount, currency)
- If any critical field has confidence < 0.7, call `askHuman` tool to confirm/correct
- Store confidence scores in ClaimState for audit trail

**Warning signs:**
- Submitted claims with "unknown" merchant or 0.0 amounts
- No clarification messages in Chainlit despite blurry receipts

### Pitfall 3: Foreign Currency Not Converted

**What goes wrong:** Receipt in USD submitted with totalAmount=50.00, currency=USD. Policy validation fails because limit is in SGD.

**Why it happens:** Currency conversion tool not called before policy validation, or conversion result not persisted.

**How to avoid:**
- In `extractReceiptFields` tool, always check if currency != "SGD"
- If foreign currency detected, immediately call `convertCurrency` tool
- Store BOTH original (USD 50.00) and converted (SGD 67.50) amounts in ClaimState
- Pass converted SGD amount to `searchPolicies` tool for validation

**Warning signs:**
- Policy violations for USD receipts that are actually under SGD limit
- Claims table has currency="USD" but no converted amount

### Pitfall 4: Chainlit Image Upload Not Handled

**What goes wrong:** User uploads receipt image in Chainlit, but `onMessage` receives no image data.

**Why it happens:** Chainlit file attachments require special handling, not automatically in `message.content`.

**How to avoid:**
- Check `message.elements` for file attachments (images are in `elements` list, not `content`)
- Use `cl.AskFileMessage` to explicitly request image upload if needed
- Enable spontaneous file upload in `config.toml` for attach-to-any-message behavior

**Warning signs:**
- User says "I uploaded an image" but agent responds "No image detected"
- `message.content` is empty when image attached

### Pitfall 5: ChatOpenAI Tool Calling Doesn't Work Through OpenRouter

**What goes wrong:** Agent calls tool, but tool never executes. Graph hangs or errors.

**Why it happens:** Not all OpenRouter models support tool calling. Free models may have limited support.

**How to avoid:**
- Check [OpenRouter tool-calling models list](https://openrouter.ai/collections/tool-calling-models) before selecting LLM
- For OPENROUTER_MODEL_LLM, use a model with "tool calling" capability (e.g., llama-3.1 models)
- Test tool calling in isolation before integrating with agent
- Set `model_kwargs={"tool_choice": "auto"}` if needed

**Warning signs:**
- Tools defined but never called, agent always returns text-only responses
- Error: "Tool calling not supported by this model"

### Pitfall 6: Database Schema Doesn't Support Dual Currency

**What goes wrong:** Alembic migration adds `convertedAmountSgd` column to `claims` table, but not to `receipts` table. Foreign receipts can't be stored correctly.

**Why it happens:** Incomplete migration, only updated claims table.

**How to avoid:**
- Add dual currency fields to BOTH `claims` and `receipts` tables:
  - `originalCurrency` (String(3))
  - `originalAmount` (Numeric(10, 2))
  - `convertedAmountSgd` (Numeric(10, 2))
- Keep existing `currency` and `totalAmount` fields for backward compatibility (set to SGD amounts)
- Update ORM models to match schema

**Warning signs:**
- Foreign currency receipts fail to persist with "column does not exist" error
- Can store claim in USD but receipt must be in SGD

### Pitfall 7: MCP Server URL Pattern Wrong

**What goes wrong:** MCP client calls fail with 404 or connection refused.

**Why it happens:** Incorrect endpoint path. FastMCP servers expose `/mcp/` not `/mcp` (trailing slash matters).

**How to avoid:**
- Always use `http://{host}:{port}/mcp/` (with trailing slash)
- Store full URLs in .env.local, not just host:port
- Test MCP endpoint with curl before integrating: `curl http://localhost:8001/mcp/`

**Warning signs:**
- httpx.HTTPStatusError: 404 when calling MCP tools
- Connection refused despite Docker container running

### Pitfall 8: Laplacian Variance Threshold Too Strict

**What goes wrong:** Sharp, clear receipt images rejected as "blurry" by quality check.

**Why it happens:** Threshold set too high (e.g., 1000) or image preprocessing affects variance.

**How to avoid:**
- Start with threshold=100, test with sample receipts
- Collect 10 sharp + 10 blurry receipts, compute variance for each, plot distribution
- Set threshold between clusters (e.g., if sharp images are 200-500 and blurry are 20-80, use 150)
- Optionally apply Gaussian blur before Laplacian to reduce noise sensitivity

**Warning signs:**
- Users complain "clear images are being rejected"
- Variance scores for rejected images are > 200

### Chainlit Image Upload Handling

```python
# Source: Chainlit documentation + community examples
import chainlit as cl
from langchain_core.messages import HumanMessage
import base64

@cl.on_message
async def onMessage(message: cl.Message):
    """Handle messages with optional image attachments."""
    graph = cl.user_session.get("graph")
    threadId = cl.user_session.get("thread_id")

    # Check for image attachments
    imageBytes = None
    if message.elements:
        for element in message.elements:
            if element.type == "image" or element.mime.startswith("image/"):
                # Read image file
                imageBytes = element.content  # bytes or path
                if isinstance(imageBytes, str):
                    with open(imageBytes, "rb") as f:
                        imageBytes = f.read()
                break

    # Encode image to base64 if present
    imageB64 = None
    if imageBytes:
        imageB64 = base64.b64encode(imageBytes).decode('utf-8')
        await cl.Message(content="Receipt image received. Processing...").send()

    # Create message for graph
    if imageB64:
        # Multimodal message (text + image)
        humanMsg = HumanMessage(content=[
            {"type": "text", "text": message.content or "Process this receipt"},
            {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{imageB64}"}}
        ])
    else:
        # Text-only message
        humanMsg = HumanMessage(content=message.content)

    # Invoke graph
    initialState = {
        "claimId": cl.user_session.get("claim_id", str(uuid.uuid4())),
        "status": "draft",
        "messages": [humanMsg],
    }

    result = await graph.ainvoke(
        initialState,
        config={"configurable": {"thread_id": threadId}}
    )

    # Handle interrupts (clarification requests)
    if "__interrupt__" in result:
        interrupt_payload = result["__interrupt__"][0].value
        await cl.Message(content=interrupt_payload["question"]).send()
        await cl.Message(content=f"Extracted data: {interrupt_payload['data']}").send()

        # Wait for user response (next message will resume)
        cl.user_session.set("awaiting_clarification", True)
        return

    # Send agent responses
    for msg in result["messages"][1:]:  # Skip HumanMessage
        await cl.Message(content=msg.content).send()
```

from mcp.client.streamable_http import streamablehttp_client
from mcp import ClientSession
import httpx

async def mcpCallTool(serverUrl: str, toolName: str, args: dict) -> dict:
    """Call MCP tool via Streamable HTTP."""
    httpClient = httpx.AsyncClient(timeout=30.0)

    async with streamablehttp_client(serverUrl, http_client=httpClient) as (read, write, _):
        async with ClientSession(read, write) as session:
            await session.initialize()
            result = await session.call_tool(toolName, args)
            return result.content

### Laplacian Variance Blur Detection

```python
# Source: PyImageSearch + Medium tutorials
import cv2
import numpy as np

def checkImageQuality(imageBytes: bytes, threshold: float = 150.0) -> dict:
    """Check if image is suitable for VLM extraction."""
    # Decode
    nparr = np.frombuffer(imageBytes, np.uint8)
    img = cv2.imdecode(nparr, cv2.IMREAD_COLOR)

    # Grayscale
    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

    # Laplacian variance
    laplacian = cv2.Laplacian(gray, cv2.CV_64F)
    variance = laplacian.var()

    # Check dimensions (reject very small images)
    height, width = gray.shape
    isLowRes = width < 800 or height < 600

    isBlurry = variance < threshold

    return {
        "acceptable": not isBlurry and not isLowRes,
        "variance": variance,
        "resolution": {"width": width, "height": height},
        "reason": "Image is blurry" if isBlurry else "Resolution too low" if isLowRes else "OK"
    }
```
