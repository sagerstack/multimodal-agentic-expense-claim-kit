The openrouter client currently has ZERO logging. It's marked DEPRECATED but still has tests and may be called as a fallback. Add structured logging for LLM call lifecycle events.

1. Add imports:
   ```python
   import logging
   import time
   from agentic_claims.core.logging import logEvent
   ```

2. Add logger:
   ```python
   logger = logging.getLogger(__name__)
   ```

3. In `callLlm()`, wrap the retry loop with logging:
   - Before the loop: `logEvent(logger, "llm.call_started", logCategory="llm", model=model_name, messageCount=len(messages))`
   - On success (after getting response, before return): `logEvent(logger, "llm.call_completed", logCategory="llm", model=model_name, elapsed=f"{time.time() - t0:.2f}s", attempt=attempt + 1)`
   - In the except block (when retrying, not the final failure): `logEvent(logger, "llm.call_retrying", logCategory="llm", model=model_name, attempt=attempt + 1, maxRetries=self.settings.openrouter_max_retries, error=str(e), level=logging.WARNING)`
   - In the except block (final attempt failure): `logEvent(logger, "llm.call_failed", logCategory="llm", model=model_name, attempt=attempt + 1, maxRetries=self.settings.openrouter_max_retries, error=str(e), elapsed=f"{time.time() - t0:.2f}s", level=logging.ERROR)`

4. Add `t0 = time.time()` before the retry loop to capture total elapsed time.

5. Do NOT add logging to `callVlm()` -- it delegates to `callLlm()` which already has the logging. Adding to both would double-log.
