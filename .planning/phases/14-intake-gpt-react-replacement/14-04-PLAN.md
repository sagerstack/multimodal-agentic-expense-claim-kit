But wait — for side questions, we want the LLM to respond (go to reasonNode), not finalizeTurnNode directly. The correct routing for side_question from interruptResolutionNode is: `sideQuestionResponderNode` if we want to add prep logic, otherwise directly to `reasonNode`.

- Route `interruptResolutionNode` to `sideQuestionResponderNode` when `lastResolution.outcome == "side_question"`

Correct topology for side_question:
1. `interruptResolutionNode` classifies → `side_question` outcome → route to `reasonNode` (LLM handles the response + re-presents the original question)
2. `sideQuestionResponderNode` is unused for now — remove it from the graph OR repurpose it

Decision: Remove `sideQuestionResponderNode` entirely from the graph. It is dead code with no semantic purpose. The side-question handling is: interruptResolutionNode classifies → reasonNode (LLM responds and re-presents) → finalizeTurnNode.

```python
from agentic_claims.core.config import getSettings

def _getIntakeGptTokenSets():
    """Load token sets from config at call time (avoids import-time settings load)."""
    settings = getSettings()
    endTokens = set(t.strip() for t in settings.intake_gpt_end_conversation_tokens.split(",") if t.strip())
    affirmativeTokens = set(t.strip() for t in settings.intake_gpt_affirmative_tokens.split(",") if t.strip())
    negativeTokens = set(t.strip() for t in settings.intake_gpt_negative_tokens.split(",") if t.strip())
    currencySymbolMap = dict(
        (pair.split(":", 1)[0].strip().lower(), pair.split(":", 1)[1].strip())
        for pair in settings.intake_gpt_currency_symbol_map.split(",")
        if ":" in pair
    )
    validCategories = set(t.strip() for t in settings.intake_gpt_valid_categories.split(",") if t.strip())
    return endTokens, affirmativeTokens, negativeTokens, currencySymbolMap, validCategories
```

- `_classifyInterruptReply` — uses all 3 token sets: call `_END_CONVERSATION_TOKENS, _AFFIRMATIVE_TOKENS, _NEGATIVE_TOKENS, _, _ = _getIntakeGptTokenSets()` at function start
