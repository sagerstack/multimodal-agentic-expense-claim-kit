<task type="auto">
  <name>Task 1: Finalize button partial styling + accessibility</name>
  <files>templates/partials/interrupt_buttons.html</files>
  <action>
Refine the partial created in Plan 14-04 with final styling and a11y attributes. Final form:

```html
{# Button-style interrupt prompt (Phase 14).
   Server emits this via SseEvent.INTERRUPT when payload.uiKind == "buttons".
   The container #interruptTarget in chat.html (sse-swap="interrupt",
   hx-swap="innerHTML") replaces its children with this partial on arrival.

   Each button is its own HTMX form so a click triggers a POST without
   bundling the textarea value. Both `message` and `button_value` carry
   the same canonical reply — `message` is the text the graph classifier
   sees; `button_value` is metadata for logging. #}
<div class="space-y-3" role="group" aria-label="Confirmation response">
  {% if question %}
  <div class="text-sm text-on-surface leading-relaxed">{{ question }}</div>
  {% endif %}
  <div class="flex gap-2 flex-wrap">
    {% for opt in options %}
    <form hx-post="/chat/message"
          hx-swap="none"
          hx-encoding="multipart/form-data"
          hx-on::after-request="document.getElementById('interruptTarget').innerHTML=''"
          class="inline-flex">
      <input type="hidden" name="button_value" value="{{ opt.value }}">
      <input type="hidden" name="message" value="{{ opt.value }}">
      <button type="submit"
              aria-label="{{ opt.label }}"
              class="px-5 py-2 rounded-xl bg-surface-container-lowest text-on-surface hover:bg-surface-bright active:scale-95 border border-outline-variant/30 text-sm font-medium transition-all focus:outline-none focus:ring-2 focus:ring-secondary/50">
        {{ opt.label }}
      </button>
    </form>
    {% endfor %}
  </div>
</div>
```

Notes:
- `hx-on::after-request` clears `#interruptTarget` immediately after POST so the buttons disappear and the chat visually transitions to "processing" state (HTMX afterRequest fires on 204 success). The SSE stream then delivers the next turn.
- `role="group"` + `aria-label` + per-button `aria-label` cover a11y basics.
- Button classes reuse the existing design system tokens (`surface-container-lowest`, `on-surface`, `outline-variant`, `secondary` ring) — no new colour values.
- `flex-wrap` allows Yes/No to stack on narrow screens.

  <name>Task 2: Disable text input while a button-type interrupt is active (optional enhancement)</name>
  <files>templates/chat.html</files>
  <action>
Small UX guard: when the `#interruptTarget` contains a button form (i.e. innerHTML is non-empty AND contains `button_value`), visually soften the textarea to signal "please click a button". Implement via a single Alpine attribute.

Add a data flag tracked by Alpine. The existing `chatState()` Alpine component (chat.html line 177) can track `buttonInterruptActive`.

1. In `chatState()`, add `buttonInterruptActive: false` to the returned object.

2. After the existing `<div id="interruptTarget" ...>` element (chat.html line 99), add a lightweight observer that sets the flag when the target's innerHTML contains a button form:

```html
<div id="interruptTarget" sse-swap="interrupt" hx-swap="innerHTML"
     hx-on::after-swap="window.dispatchEvent(new CustomEvent('stream-done')); window.dispatchEvent(new CustomEvent('interrupt-swapped', {detail: {hasButtons: this.querySelector('button[aria-label=\\'Yes\\'], button[aria-label=\\'No\\']') !== null}}))"
     class="max-w-2xl text-sm text-on-surface p-4 bg-surface-container-low rounded-2xl border border-outline-variant/5 shadow-xl empty:hidden"></div>
```

3. In the Alpine `x-data="chatState()"` root element (chat.html `section` tag around line ~45, or wherever chatState is attached), add:

```html
@interrupt-swapped.window="buttonInterruptActive = $event.detail.hasButtons"
@stream-done.window="buttonInterruptActive = false"
```

4. Update the existing textarea input (chat.html around line 131) to visually dim when `buttonInterruptActive`:

```html
<textarea name="message" x-model="messageText" x-ref="messageInput"
          :class="{'opacity-50': buttonInterruptActive}"
          :placeholder="buttonInterruptActive ? 'Please click Yes or No above' : 'Type a message or drop files...'"
          class="flex-1 bg-transparent border-none focus:ring-0 focus:outline-none outline-none text-on-surface placeholder:text-outline/50 py-3 resize-none font-body text-sm"
          style="outline: none !important;"
          rows="1"
          @keydown.enter.prevent="if (!$event.shiftKey) $el.form.requestSubmit()"></textarea>
```

Do NOT use `:disabled` — the user should still be able to type a free-text side question. The placeholder hint + opacity signal "buttons are the primary path".
