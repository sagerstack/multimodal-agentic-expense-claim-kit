# Phase 6.3: User Authentication + Dual Roles + Reviewer Pages - Context

**Gathered:** 2026-04-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Add user authentication with dual roles (user/reviewer) and implement all 3 remaining Stitch-designed pages: Approver Dashboard, Audit Log, and Claim Review. This absorbs the scope of Phase 8 (Dashboard + Audit Log) and Phase 9 (Claim Review) into Phase 6.3. The Claims (Chat) page is frozen — no changes in this phase.

</domain>

<decisions>
## Implementation Decisions

### Navigation Structure
- Sidebar has exactly 2 entries: **Claims** (both roles, → `/`) and **Review** (reviewer only, → `/dashboard`)
- Audit Log page (`/audit/{claim_id}`) is NOT in the sidebar — accessed via dashboard row audit icon
- Claim Review page (`/review/{claim_id}`) is NOT in the sidebar — accessed via dashboard row click
- Remove mobile bottom nav and floating action button — desktop only

### Login & Authentication
- Login page matches Stitch design (`docs/ux/stitch-login-expense-ai.html`) with dark theme, glass panel, neon glow
- "Remember this device" checkbox is functional — extends session cookie expiry (e.g., 7 days vs session-only)
- Failed login shows inline error message below form fields ("Invalid username or password"), stays on login page
- Logout is in the top nav — click profile avatar to reveal dropdown with Logout option
- After login, redirect to role default: user → `/`, reviewer → `/dashboard`. No "return to original URL" behavior.
- Password hashing: Claude's discretion (passlib/bcrypt or argon2)
- Session storage: Claude's discretion (existing cookie session vs server-side)

### Seed Data
- 4 users seeded via Alembic migration (same migration that creates users table)
- Users table includes `display_name` field

| josiah   | Josiah      | user     | 1010740     | josiah123 |
| james    | James       | reviewer | 909090      | james123  |
| tung     | Tung        | reviewer | 909091      | tung123   |

### Dashboard (Approver Dashboard)
- 3 KPI cards: Total Pending, Auto-Approved, Escalated — count numbers only, no context badges (no "+12% vs last wk")
- AI Efficiency chart: real chart with hourly auto-approval rates calculated from claims table
- Auto-Approval Threshold: read-only display from config, not editable
- Filter and Export buttons: removed — show all claims in table
- Claims table columns: Employee (display_name + employee_id), Category, Amount, Status badge, Action column
- Row click → navigates to `/review/{claim_id}` (Claim Review page)
- Row last column: audit icon/link → navigates to `/audit/{claim_id}` (Audit Log page)

### Audit Log (Audit & Transparency)
- List-detail layout matching Stitch design
- Left panel: full claim list with the navigated claim pre-selected, HTMX partial swap on card click (right panel only)
- Full list with status filter via HTMX
- Decision timeline: 4 steps (Receipt Uploaded → AI Extraction → Policy Check → Final Decision)
- Add intake step logging to the intake agent — write audit_log entries for upload, extraction, policy check (real timeline data)
- Confidence score: parsed from `intake_findings` JSONB on the claims table
- Intelligence Insights: real data — Anomaly = violations count from intake_findings, Cost Benchmark = compare claim amount to category average
- View Receipt button: functional (opens receipt image). Download Log button: removed.
- Branding: "Expense AI" (not "Cognitive Atelier" / "Luminous Intelligence")

### Claim Review
- Split view: Original Receipt (left) + Extracted Entity Data (right), matching Stitch layout
- All extracted fields are read-only — no editing by reviewer
- Flag Reason card: real data from `intake_findings` JSONB, with fallback ("No assessment data available")
- AI Insight card: real data — query employee's past claims for submission count + approval rate, with fallback
- Rejection reasons: radio buttons (Duplicate submission, Incomplete receipt details, Policy violation) — must select before Reject activates
- Reviewer notes: free-text textarea
- After Approve or Reject: redirect back to `/dashboard`
- Reassign button: removed
- View History button: removed
- Receipt zoom: Alpine.js CSS transform scale (as per original Phase 9 spec)

### Claims Page
- **FROZEN** — no changes in this phase
- Employee ID continues with existing server-side extraction from chat messages
- No auth-based employee ID replacement in this phase

### Data & Schema Changes
- New `users` table: username, hashed_password, role (user/reviewer), employee_id, display_name
- Alembic migration `003_add_users_table.py` creates table + seeds 4 users
- `intake_findings` JSONB already exists on claims table (written by DB MCP server) — no schema change needed
- Add intake step audit logging (new audit_log entries during intake agent processing)

### Branding
- Keep "Expense AI" throughout all pages
- Do not adopt Stitch branding ("Cognitive Atelier", "Luminous Intelligence")

### Claude's Discretion
- Password hashing library choice
- Session storage mechanism
- HTMX swap strategies for partial updates
- Chart rendering approach for AI Efficiency
- Exact receipt zoom bounds and increment

</decisions>
