  <action>
    1. **Add `insertAuditLog` tool to the DB MCP server** (`mcp_servers/db/server.py`):
       The existing `executeQuery` tool is SELECT-only for safety. Add a dedicated write tool:
       ```python
       @mcp.tool()
       def insertAuditLog(claimId: int, action: str, newValue: str, actor: str, oldValue: str = "") -> dict:
           conn = getConnection()
           with conn.cursor() as cur:
               cur.execute(
                   "INSERT INTO audit_log (claim_id, action, old_value, new_value, actor) VALUES (%s, %s, %s, %s, %s) RETURNING id, timestamp",
                   (claimId, action, oldValue, newValue, actor)
               )
               conn.commit()
               row = cur.fetchone()
               return {"id": row[0], "timestamp": row[1].isoformat()}
       ```

    2. **Create `src/agentic_claims/agents/intake/auditLogger.py`** with the buffer + flush pattern:
       - Module-level `_auditBuffer: dict[str, list[dict]]` keyed by session claimId (string, the session-scoped claim identifier -- not the DB int ID which doesn't exist yet during intake).
       - `bufferStep(sessionClaimId: str, action: str, details: dict) -> None`: Appends `{action, details, bufferedAt: datetime.utcnow().isoformat()}` to `_auditBuffer[sessionClaimId]`. Creates the list if not present.
       - `async flushSteps(sessionClaimId: str, dbClaimId: int, actor: str = "intake_agent") -> None`: Iterates `_auditBuffer[sessionClaimId]`, calls the DB MCP server's `insertAuditLog` tool for each entry (via mcpClient HTTP POST). After flushing, deletes the buffer entry for that sessionClaimId. Handle errors gracefully: log warning per failed entry but continue flushing remaining entries. Never raise -- audit logging must not crash the intake pipeline.
       - `async logIntakeStep(claimId: int, action: str, details: dict, actor: str = "intake_agent") -> None`: Direct (non-buffered) write for cases where DB claim ID is already known. Calls `insertAuditLog` via MCP. Wraps in try/except, logs warning on failure.

    3. **Integrate into `extractReceiptFields.py`**:
       - AFTER successful VLM extraction (after the extraction result is parsed), buffer two audit steps:
         ```python
         bufferStep(sessionClaimId=claimId, action="receipt_uploaded",
                    details={"imagePath": imagePath, "imageQuality": qualityResult})
         bufferStep(sessionClaimId=claimId, action="ai_extraction",
                    details={"confidence": confidence, "merchant": merchant, "amount": amount, "fields": extractedFields})
         ```
       - The `claimId` here is the session-scoped string (from ClaimState), not the DB int.

    4. **Integrate into `searchPolicies.py`**:
       - AFTER policy search returns results, buffer a policy_check audit step:
         ```python
         bufferStep(sessionClaimId=claimId, action="policy_check",
                    details={"violations": violations, "policyRefs": policyReferences, "compliant": len(violations) == 0})
         ```

    5. **Integrate into `submitClaim.py`**:
       - AFTER the claim is successfully inserted via the DB MCP server (which returns the DB claim ID), flush all buffered audit steps:
         ```python
         await flushSteps(sessionClaimId=sessionClaimId, dbClaimId=dbClaimId)
         ```
       - The submitClaim tool already has access to both the session claimId and the returned DB claim ID.
