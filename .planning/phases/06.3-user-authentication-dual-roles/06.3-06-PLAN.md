    1. Create `src/agentic_claims/agents/intake/auditLogger.py`:
       - `async def logIntakeStep(claimId: int, action: str, details: dict, actor: str = "intake_agent") -> None`:
         - Calls the DB MCP server to INSERT into audit_log table
         - Uses the existing mcpClient pattern: HTTP POST to the DB MCP server's `executeQuery` tool (or a new dedicated tool if cleaner)
         - Parameters:
           - claimId: the DB claim ID (int)
           - action: one of "receipt_uploaded", "ai_extraction", "policy_check"
           - details: dict that gets JSON-serialized into `new_value` column
           - actor: defaults to "intake_agent"
         - The MCP DB server already has an `executeQuery` tool that can run INSERT statements (check if it's SELECT-only; if so, add an `insertAuditLog` tool to the DB MCP server)
         - Handle errors gracefully: log warning but don't fail the intake pipeline if audit logging fails (audit is observability, not critical path)

    2. **IMPORTANT**: Check if `executeQuery` in the DB MCP server is SELECT-only. If yes, either:
       a. Add a new `insertAuditLog` MCP tool to `mcp_servers/db/server.py` that accepts (claimId, action, oldValue, newValue, actor) and does the INSERT, OR
       b. Relax `executeQuery` to allow INSERT (less safe but simpler)
       Option (a) is preferred for safety. The tool should:
       ```python
       @mcp.tool()
       def insertAuditLog(claimId: int, action: str, newValue: str, actor: str, oldValue: str = "") -> dict:
           conn = getConnection()
           with conn.cursor() as cur:
               cur.execute(
                   "INSERT INTO audit_log (claim_id, action, old_value, new_value, actor) VALUES (%s, %s, %s, %s, %s) RETURNING id, timestamp",
                   (claimId, action, oldValue, newValue, actor)
               )
               conn.commit()
               row = cur.fetchone()
               return {"id": row[0], "timestamp": row[1].isoformat()}
       ```
