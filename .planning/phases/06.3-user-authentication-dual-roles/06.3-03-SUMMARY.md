## What Was Implemented

### src/agentic_claims/web/db.py (new)
- Shared `getAsyncSession()` async context manager using `create_async_engine` + `async_sessionmaker`
- Reusable by all web routers — avoids duplicate engine creation

### src/agentic_claims/web/routers/dashboard.py (new)
- `GET /dashboard`: HTML page handler (moved from pages.py), queries KPIs + claims server-side, reviewer-only
- `GET /api/dashboard/kpis`: JSON `{pending, autoApproved, escalated}` counts from DB
- `GET /api/dashboard/claims`: JSON list with employeeName (user join), status, amount, currency
- `GET /api/dashboard/efficiency`: JSON `{hourlyRates: [{hour, rate}]}` from last 24h claims data
- All endpoints return 403 for non-reviewer role

### src/agentic_claims/web/routers/pages.py
- Removed `/dashboard` route (now owned by dashboard.py)

### templates/dashboard.html
- 3 KPI cards: Total Pending (primary), Auto-Approved (secondary), Escalated (tertiary)

- Status badges: color-coded per status (pending=primary, approved=secondary, escalated=tertiary, rejected=error)
- AI Efficiency section with JS chart fetching `/api/dashboard/efficiency`
- Auto-Approval Threshold display (read-only, SGD 500.00)

### templates/partials/claims_table_rows.html (new)
- Table row partial for claims list with status badges
- Handles empty state with "No claims to display" message
