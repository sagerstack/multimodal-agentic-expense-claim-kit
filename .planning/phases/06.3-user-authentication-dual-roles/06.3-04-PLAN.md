  <files>

  <action>
    1. Create `src/agentic_claims/web/routers/audit.py`:

       - `GET /audit/{claimId}` -> **HTML page handler** (moved from pages.py). This serves the audit page with full server-side data:
         - Import `getCurrentUser` from auth.py, check `role == "reviewer"` (redirect to `/` if not)
         - Query the claim, its audit_log timeline entries, insights data, and the full claims list for the left panel -- all server-side
         - Pass to template context: `claim`, `timelineSteps`, `insights`, `allClaims`, `selectedClaimId`, `userRole`, `displayName`, `employeeId`, `username`, `activePage="audit"`
         - Render `templates/audit.html`

       - `GET /api/audit/{claimId}/timeline` -> returns JSON timeline data:
         Query audit_log entries for the claim, ordered by timestamp ASC.
         Map audit_log actions to 4 timeline steps:
         - "receipt_uploaded" action -> Receipt Uploaded step
         - "ai_extraction" action -> AI Extraction step (parse confidence from new_value JSON)
         - "policy_check" action -> Policy Check step (parse policy reference from new_value JSON)
         - "claim_submitted" / "status_change" action -> Final Decision step
         Return: `{steps: [{name, status, timestamp, details, confidence?, policyRef?}]}`
         If fewer than 4 audit entries exist, show available steps as completed and remaining as pending.

       - `GET /api/audit/{claimId}/insights` -> returns JSON:
         - `anomalyCount`: Count of violations from the claim's intake_findings JSONB (if exists, else 0)
         - `costBenchmark`: Compare this claim's total_amount to the average of claims in the same category. Return `{claimAmount, categoryAverage, category}`
         - Parse intake_findings from the claims table (it may contain `{violations: [...], confidence: float}`)

       - `GET /api/audit/claims` -> returns JSON list of all claims for the left panel:
         `{id, claimNumber, status, totalAmount, currency, createdAt}` ordered by created_at DESC

       - `GET /api/audit/{claimId}/receipt` -> returns the receipt image:
         Query the receipt record for the claim, get image_path. Serve the file. If no image, return 404.

    2. **Remove the `/audit/{claimId}` route from `src/agentic_claims/web/routers/pages.py`**. The audit.py router now owns this route entirely. Leave all other routes in pages.py untouched.

    3. Include the audit router in main.py (`app.include_router(auditRouter)`).

    4. Create `tests/test_audit.py`:
       - `testTimelineEndpointReturnsSteps`: Mock audit_log query, verify 4-step structure
       - `testTimelineEndpointHandlesMissingSteps`: No audit entries -> all steps pending
       - `testInsightsEndpointReturnsAnomalyAndBenchmark`: Mock claims query with intake_findings
       - `testClaimsListEndpointReturnsSortedList`: Verify DESC order
  </action>
  <verify>
    `poetry run pytest tests/test_audit.py -v` -- all tests pass.
    With app running and at least 1 claim with audit entries: `curl http://localhost:8000/api/audit/1/timeline` returns timeline JSON.
    Login as reviewer, navigate to `/audit/1` -- page loads with full data (not empty template).
  </verify>
  <done>Audit API endpoints return timeline data from audit_log, insights from intake_findings, and claim list. Audit HTML page handler moved from pages.py to audit.py with full server-side data loading.</done>
