# Plan 06.3-06 Summary: Intake Audit Logging

## Tasks Completed

### Task 1: Audit logging helper, insertAuditLog MCP tool, integration into intake tools

**`mcp_servers/db/server.py`** — Added `insertAuditLog` tool:
- Parameters: `claimId`, `action`, `newValue`, `actor`, `oldValue` (optional)
- Inserts directly into `audit_log` table, returns `{id, timestamp}`
- `executeQuery` remains SELECT-only (unchanged)

**`src/agentic_claims/agents/intake/auditLogger.py`** — New module with buffer + flush pattern:
- `_auditBuffer: dict[str, list[dict]]` — module-level buffer keyed by session claimId
- `bufferStep(sessionClaimId, action, details)` — appends entry with `bufferedAt` timestamp
- `flushSteps(sessionClaimId, dbClaimId, actor)` — flushes all buffered entries via MCP `insertAuditLog`, clears buffer, never raises
- `logIntakeStep(claimId, action, details, actor)` — direct (non-buffered) write for when DB ID is known, never raises

**`tools/extractReceiptFields.py`** — Buffers 2 steps after successful VLM extraction:
- `receipt_uploaded`: `{imagePath, imageQuality}`
- `ai_extraction`: `{confidence, merchant, amount, fields}`

**`tools/searchPolicies.py`** — Added optional `claimId` parameter; buffers 1 step after policy search:
- `policy_check`: `{violations, policyRefs, compliant, query}`

**`tools/submitClaim.py`** — Added optional `sessionClaimId` parameter; flushes buffered steps after successful claim insertion:
- Calls `flushSteps(sessionClaimId, dbClaimId)` once DB claim ID is returned from MCP
