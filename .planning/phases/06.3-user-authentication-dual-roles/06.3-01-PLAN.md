    1. Create `src/agentic_claims/web/auth.py` with:
       - `authenticateUser(username: str, password: str) -> User | None`: Queries users table by username, verifies password with passlib bcrypt. Returns User object or None. Uses a new async SQLAlchemy session (create an `getDbSession()` async context manager using the postgres_dsn_async from Settings, or reuse the checkpointer pool from app.state if simpler).
       - `requireAuth(request: Request) -> RedirectResponse | None`: Checks `request.session` for `user_id`. If missing, returns `RedirectResponse("/login", status_code=302)`. This will be used as a dependency or called at the start of each protected route.
       - `getCurrentUser(request: Request) -> dict`: Returns `{userId, username, role, employeeId, displayName}` from session. Raises 401 if no session.
       - `requireRole(role: str)`: Returns a dependency that checks session role matches. If not, redirects user role to `/`, reviewer to `/dashboard`.

    2. Create `src/agentic_claims/web/routers/auth.py` with:
       - `GET /login`: Renders login.html. If already authenticated, redirect to role default (user -> `/`, reviewer -> `/dashboard`).
       - `POST /login`: Accepts form data (username, password, remember_me checkbox). Calls `authenticateUser`. On success: set session keys (user_id, username, role, employee_id, display_name), set cookie max_age to 604800 (7 days) if remember_me else None (session-only), redirect to role default. On failure: re-render login.html with error="Invalid username or password" and the submitted username preserved.
       - `GET /logout`: Clear session, redirect to `/login`.

    3. Update `src/agentic_claims/web/main.py`:
       - Import and include authRouter
       - Add auth middleware: Create a Starlette middleware (or use FastAPI middleware) that checks every request except `/login`, `/logout`, `/static/*` paths. If no `user_id` in session, redirect to `/login`. This ensures ALL pages require auth without modifying each route handler individually.

    4. Create `templates/login.html`:

       - Use existing Tailwind classes and color tokens from base.html (bg-surface, text-on-surface, from-primary, etc.)
       - Do NOT extend base.html -- login is a standalone page (no sidebar, no top nav)
       - Include error message div (conditionally shown) below the form fields: red text "Invalid username or password"
       - Preserve username in the form field on failed login attempt via Jinja2 `{{ username }}`

       - `testLoginGetRendersForm`: Test GET /login returns 200
