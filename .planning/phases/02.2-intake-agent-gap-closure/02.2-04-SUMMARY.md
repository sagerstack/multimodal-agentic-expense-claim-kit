## What Was Built

Logs now flow to stdout (JSON via python-json-logger) AND to Seq (CLEF via HTTP).

## Technical Notes

### Docker Networking for Seq

- **Browser URL (seq_url)**: http://localhost:5341 — Maps to Seq's internal port 80
- **Docker-internal URL (seq_ingestion_url)**: http://seq/api/events/raw — App uses this for HTTP POST
- Port mapping: 5341:80 (external:internal) and 5342:5341 (ingestion API in docs, but actual endpoint is /api/events/raw on port 80 internally)

### CLEF Format

Seq expects one JSON object per line:

```json
{
  "@t": "2026-03-26T07:15:58.383Z",
  "@l": "Information",
  "@mt": "Chat session started - thread_id: {ThreadId}, claim_id: {ClaimId}",
  "logger": "agentic_claims.app",
  "function": "onChatStart",
  "line": 44,
  "service": "agentic-claims",
  "environment": "local"
}
```

### Logging Architecture

1. **Console/File**: python-json-logger formatter → stdout/file
2. **Seq**: SeqHandler formats CLEF directly → HTTP POST to Seq
3. **Centralized**: setupLogging() in core/logging.py configures all handlers
4. **Per-session**: app.py calls setupLogging() in onChatStart() (once per chat session)
