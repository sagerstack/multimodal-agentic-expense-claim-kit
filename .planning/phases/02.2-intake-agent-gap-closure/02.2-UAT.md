### 4. Collapsible Chain-of-Thought
expected: During agent processing, tool reasoning and internal messages are NOT visible in the main chat. They appear in collapsed/collapsible Step elements (expandable if clicked). User only sees the agent's step announcements.
result: issue
reported: "CoT is not being recorded in the Step element — just a single static sentence ('Handling user response...' / 'Receipt processing complete'). Actual tool calls, LLM reasoning, and intermediate messages are not captured. Step is a label, not a CoT container."
severity: minor

### 5. Description-Receipt Cross-Reference
expected: Start a conversation saying "I need to submit a hotel expense" then upload a restaurant receipt. At Step 3, the agent flags the contradiction between your description ("hotel") and the receipt data ("restaurant") and asks for clarification.
result: issue
reported: "Cross-reference works but the design is wrong: (1) This step should be OPTIONAL — only if user provided a description. No description = use receipt as source and skip. (2) Fundamental UX issue: ALL steps should be internal CoT, not shown explicitly to user as 'Step 1:', 'Step 2:' etc. User should see conversational output — a summary of what was done, what's next. Numbered steps are internal workings."
severity: major
