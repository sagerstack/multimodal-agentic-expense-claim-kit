## Seq CLEF ingestion
Seq accepts Compact Log Event Format (CLEF) via HTTP POST to /api/events/raw. Each line is a JSON object with fields:
- @t: ISO 8601 timestamp
- @l: level (Information, Warning, Error, etc.)
- @mt: message template
- @x: exception (optional)
- Additional properties are indexed automatically.

The handler should batch events or send them individually. For simplicity, send each log record as it occurs (no batching needed for dev).

## Docker networking
Inside the Docker network, the app service reaches Seq at http://seq:5341 (internal port 80 mapped from 5341). The /api/events/raw endpoint accepts POST with Content-Type: application/vnd.serilog.clef. BUT wait — Seq's internal port is 80, and the Docker port mapping is 5341:80. So inside Docker, the URL is http://seq:80/api/events/raw OR http://seq/api/events/raw.

IMPORTANT: seq_url in .env.local is "http://localhost:5341" (for browser). For the app running inside Docker, we need a SEPARATE setting: seq_ingestion_url = "http://seq/api/events/raw" (Docker-internal). Add this to Settings and .env.local.
