  <name>Task 1: Create startup.sh automation script</name>
  <files>

  <action>
    Create `scripts/startup.sh` following the CONTEXT.md and RESEARCH.md specifications:

    **Script structure (6 numbered steps with colored output):**

    1. **Header and argument parsing:**
       - `#!/bin/bash` with `set -e`
       - Define color variables: RED, GREEN, YELLOW, NC (no color)
       - Parse `--reset` flag (sets RESET_MODE=true)
       - Print header banner with "Agentic Claims Startup Script"

    2. **Step 1/6: Stop existing containers:**
       - `docker compose down` (always — clean restart every time per CONTEXT.md)
       - Print yellow step header, green checkmark on success

    3. **Step 2/6: Handle reset mode:**
       - If `--reset`: `docker compose down -v` (wipe volumes)
       - Print what happened (volumes wiped vs kept)

    4. **Step 3/6: Start Docker Compose:**
       - `docker compose up -d --build`
       - Print step header, green checkmark on success

    5. **Step 4/6: Wait for health checks (120s timeout):**
       - Define `waitForHealthy()` function that polls `docker inspect --format='{{.State.Health.Status}}'`
       - Function parameters: service name, timeout (default 120), interval (default 5)
       - On "healthy": print green checkmark, return 0
       - On "unhealthy": print red X, show last 20 lines of logs, return 1
       - On timeout: print red X with timeout message, return 1
       - Show progress: `"\r  ⏳ $service: $status (${elapsed}s elapsed)..."`
       - Wait for each service in order: postgres, qdrant, mcp-rag, mcp-db, mcp-currency, mcp-email, seq, app
       - If any service fails: print error message with `docker compose logs $service` hint, exit 1

    6. **Step 5/6: Run Alembic migrations:**
       - `docker compose exec -T app alembic upgrade head`
       - Add retry logic: try 3 times with 2s delay between retries (migrations can fail on first attempt if DB is still warming up despite health check passing)
       - Print green checkmark on success, red X on failure

    7. **Step 6/6: Ingest policies:**
       - Run unconditionally for both reset and normal modes: `docker compose exec -T app python scripts/ingest_policies.py`
       - Note: `ingest_policies.py` has no CLI flags (no argparse). It calls `ingestPolicies()` which always deletes and recreates the Qdrant collection, making it fully idempotent. No conditional logic needed — just run it every time.
       - Print green checkmark on success

    8. **Success banner:**
       - Print green separator
       - Print service URLs: Chainlit UI (localhost:8000), Seq Logs (localhost:5341), Postgres (localhost:5432), Qdrant (localhost:6333)
       - Follow app logs: `docker compose logs -f app`

    **Make the script executable:** `chmod +x scripts/startup.sh`

    **Important implementation details:**
    - Use `docker compose` (v2 syntax, no hyphen) throughout
    - The `-T` flag on `docker compose exec` disables pseudo-TTY allocation (needed for non-interactive scripts)
    - Health check status comes from Docker inspect, not curl (Docker's own health check mechanism)
    - Script must exit with non-zero code on ANY failure (set -e handles most cases, but explicitly check health check return codes)
  </action>
