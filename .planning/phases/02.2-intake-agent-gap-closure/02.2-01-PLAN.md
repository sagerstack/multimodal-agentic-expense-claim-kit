    **Root cause context:** submitClaim currently makes two separate MCP calls (insertClaim + insertReceipt). The insertReceipt tool doesn't even exist in the DB MCP server. The CONTEXT.md decision is to merge both operations into a single atomic insertClaim MCP tool call.

    1. **Rewrite `mcp_servers/db/server.py` `insertClaim` tool**: Merge claim + receipt + intake_findings insertion into one atomic transaction.
       - New signature: `insertClaim(claimNumber, employeeId, status, totalAmount, currency, intakeFindings, receiptNumber, merchant, receiptDate, receiptTotalAmount, receiptCurrency, lineItems, imagePath, paymentMethod, taxAmount, originalAmount, originalCurrency, convertedAmount, convertedCurrency, exchangeRate, conversionDate)` — all receipt fields optional (some claims may not have receipt data at insert time; use `= None` defaults for optional params)
       - Single transaction: INSERT into claims (including intake_findings JSONB), get claim id, INSERT into receipts with FK link, COMMIT
       - On any failure: ROLLBACK both
       - RETURNING: Return dict with `{"claim": {...}, "receipt": {...}}` combining both records
       - Keep `serializeRow()` for datetime/Decimal serialization
       - Also add proper SQL column names for the new dual-currency columns from migration 002 (original_amount, original_currency, converted_amount, converted_currency, exchange_rate, conversion_date)

    2. **Rewrite `src/agentic_claims/agents/intake/tools/submitClaim.py`**: Simplify to single MCP call.
       - New signature: `submitClaim(claimData: dict, receiptData: dict, intakeFindings: dict) -> dict`
       - Merge claimData + receiptData + intakeFindings into a single `arguments` dict
       - Single `mcpCallTool(serverUrl=settings.db_mcp_url, toolName="insertClaim", arguments=mergedArgs)` call
       - Add structured logging at every step:
         - Tool entry: log input dict keys
         - Before MCP call: log serverUrl and toolName
         - After MCP call: log `type(result).__name__`, `result` if dict, first 200 chars if string
         - Handle string response: try `json.loads()`, log the conversion
         - Tool exit: log success/failure and return value keys
       - Return the dict from MCP (already has `{"claim": {...}, "receipt": {...}}` shape)

    3. **Add logging to `src/agentic_claims/agents/intake/utils/mcpClient.py`**:
       - Add `logger = logging.getLogger(__name__)` at module level
       - Log at tool entry: serverUrl, toolName, argument keys
       - Log before `session.call_tool()`: "Calling MCP tool"
       - Log after `session.call_tool()`: result content type, number of content items
       - Log at JSON parse: raw text (first 200 chars), parsed type
       - Log at function exit: return type and value summary
       - Log exceptions with full traceback: `logger.error(..., exc_info=True)`
