## Decisions Made

**System Prompt Architecture:**
- Separated into USER-FACING OUTPUT (conversational tone, no numbered steps) and INTERNAL REASONING (12-step guidance for agent)
- Cross-reference step made CONDITIONAL: only runs if user provided expense description; skips entirely if user only uploaded image
- Few-shot examples demonstrate conversational format (not numbered-step format)
- Confidence shown as High/Medium/Low buckets instead of raw scores (0.95 → "High")
- Policy check results embedded in summary card (not separate user-facing step)

**Message Handling in app.py:**
- Track message count from graph state before invoke: `messageCountBefore = len(currentState.values.get("messages", []))`
- Slice result to only new messages: `newMessages = result["messages"][messageCountBefore:]`
- Prevents re-displaying entire conversation history on each user interaction

**CoT Capture:**
- Separate new messages into `cotEntries` (ToolMessages, AI messages with tool_calls only) and `userFacingMessages` (AI messages with content)
- Write `cotEntries` to `processingStep.output` (visible in collapsed Step panel)
- Send `userFacingMessages` AFTER Step context exits (appear in main chat)
- Chainlit config.ui.cot set to "full" to show Step elements by default (user can collapse)

**HumanMessage Enhancement:**
- If user uploads image WITH text: include description in message (`User says: "..."`)
- If user uploads image WITHOUT text: explicitly note `No expense description was provided`
- Enables agent to skip conditional cross-reference step instead of stalling
