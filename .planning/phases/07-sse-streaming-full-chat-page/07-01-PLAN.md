  <name>Task 1: SSE event constants, session queue module, and ported streaming helpers</name>
  <files>

  <action>
    Create three new modules in `src/agentic_claims/web/`. Execute in this order: (1) sseEvents.py, (2) sessionQueues.py, (3) sseHelpers.py. The first two are small (~20 lines each) and take <5 min. sseHelpers.py is the bulk of the work — the 5 ported helpers are copy-paste, and runGraph is a direct port from app.py's streaming logic.

    **sseEvents.py** — Define `class SseEvent` with string constants for all SSE event types. Use the exact names from the research (lowercase-with-hyphens): `TOKEN = "token"`, `THINKING_START = "thinking-start"`, `STEP_NAME = "step-name"`, `STEP_CONTENT = "step-content"`, `THINKING_DONE = "thinking-done"`, `MESSAGE = "message"`, `SUMMARY_UPDATE = "summary-update"`, `DONE = "done"`, `ERROR = "error"`, `INTERRUPT = "interrupt"`. These constants are the single source of truth — never use inline event name strings elsewhere.

    **sessionQueues.py** — Module-level dict `_queues: dict[str, asyncio.Queue]` keyed by `threadId`. Two public functions:
    - `getOrCreateQueue(threadId: str) -> asyncio.Queue` — returns existing or creates new `asyncio.Queue(maxsize=10)`
    - `removeQueue(threadId: str) -> None` — pops from dict, no-op if missing

    **sseHelpers.py** — Port these verbatim from `src/agentic_claims/app.py` (do NOT rewrite, preserve exact logic):
    - `TOOL_LABELS` dict (getClaimSchema, extractReceiptFields, searchPolicies, convertCurrency, submitClaim)
    - `_stripToolCallJson(text: str) -> str`
    - `_stripThinkingTags(text: str) -> str`
    - `_formatElapsed(elapsed: float) -> str`
    - `_summarizeToolOutput(toolName: str, toolOutput) -> str`

    Then add a new `async def runGraph(graph, graphInput, request, templates)` generator function that:
    1. Yields `ServerSentEvent(data="", event=SseEvent.THINKING_START)`
    2. Determines fresh message vs interrupt resume from `graphInput.get("isResume")`
    3. For fresh: builds `{"claimId": ..., "status": "draft", "messages": [HumanMessage(content)]}` graph input
    4. For resume: builds `Command(resume={"response": message, "action": "confirm"})`
    5. Iterates `graph.astream_events(invokeInput, config={"configurable": {"thread_id": threadId}}, version="v2")`
    6. Classifies events exactly as app.py does: `on_chat_model_stream` -> buffer tokens + yield TOKEN events (skip during pending tool calls); `on_chat_model_end` -> capture reasoning entries; `on_tool_start` -> yield STEP_NAME; `on_tool_end` -> yield STEP_CONTENT, decrement pending
    7. Captures Type B reasoning from `chunk.additional_kwargs.get("reasoning_content")` or `chunk.additional_kwargs.get("reasoning")` and also from `chunk.response_metadata`
    8. Checks `await request.is_disconnected()` at each iteration — breaks if True (STRE-04)
    9. After stream loop: yields THINKING_DONE with summary string `"Thought for {elapsed} . {n} tools"`
    10. Checks for interrupt via `graph.aget_state()` — if `finalState.next` has tasks with interrupts, yields INTERRUPT event and sets `request.session["awaiting_clarification"] = True`
    11. Otherwise, extracts final response text (same fallback logic as app.py — state extraction if buffer empty), renders it into an HTML fragment using `templates.get_template("partials/message_bubble.html").render(content=finalText, isAi=True)`, and yields MESSAGE event
    12. Yields DONE sentinel

    Import `ServerSentEvent` from `fastapi.sse`. Import `HumanMessage` from `langchain_core.messages`. Import `Command` from `langgraph.types`. Import `SseEvent` from the sseEvents module.

    For the image-containing message, use the same text format as app.py: `'User says: "{userText}"\n\nI\'ve also uploaded a receipt image for claim {claimId}. Please process it using extractReceiptFields.'` (with image) or the no-text variant.
  </action>

  <name>Task 2: Chat router endpoints and main.py registration</name>
  <files>
    src/agentic_claims/web/routers/chat.py
    src/agentic_claims/web/main.py
  </files>
  <action>
    **chat.py** — Create `router = APIRouter()` with three endpoints:

    `POST /chat/message` — Accepts `message: str = Form(default="")` and `receipt: UploadFile | None = File(default=None)`. Gets session IDs via `getSessionIds(request)`. If `receipt` is provided, reads bytes, base64-encodes, stores via `storeImage(claimId, imageB64)`. Checks `request.session.get("awaiting_clarification", False)` to determine resume vs fresh message. Builds the graph input dict: `{"threadId": threadId, "claimId": claimId, "message": message, "hasImage": bool(receipt), "isResume": awaitingClarification}`. If resume, also sets `"resumeData": {"response": message, "action": "confirm"}` and clears the session flag. Puts input into `getOrCreateQueue(threadId)`. Returns `Response(status_code=204)`. Use `from starlette.responses import Response`.

    `GET /chat/stream` — Gets session IDs. Gets queue via `getOrCreateQueue(threadId)`. Gets graph from `request.app.state.graph`. Import `templates` from `agentic_claims.web.templating` (the shared Jinja2Templates instance). Defines `async def eventGenerator()` that loops: checks `request.is_disconnected()`, does `asyncio.wait_for(queue.get(), timeout=30.0)` (on timeout yields ping `ServerSentEvent(data="", event="ping")` and continues), then iterates `runGraph(graph, graphInput, request, templates)` yielding each SSE event. Returns `EventSourceResponse(eventGenerator())`. Import `EventSourceResponse` from `fastapi.sse`.

    `POST /chat/reset` — Gets old session IDs. Calls `clearImage(oldClaimId)` from imageStore. Calls `removeQueue(oldThreadId)` from sessionQueues. Generates new UUIDs for `thread_id` and `claim_id` in `request.session`. Removes `awaiting_clarification` from session. Returns `Response(status_code=204, headers={"HX-Redirect": "/"})`.

    **main.py** — Add import `from agentic_claims.web.routers.chat import router as chatRouter` and `app.include_router(chatRouter)` after the existing pagesRouter include.
  </action>
