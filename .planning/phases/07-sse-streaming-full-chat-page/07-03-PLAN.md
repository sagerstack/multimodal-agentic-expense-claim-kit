  <action>
    Rewrite the content block of `templates/chat.html` to be fully functional. Preserve the existing Stitch layout structure (two-column: chat + summary) and all Tailwind classes but add the interactive wiring.

    **Alpine.js state container:**
    Wrap the entire content in `<div x-data="chatState()">` where `chatState()` is defined in an inline `<script>` at the bottom (or use Alpine.data). State properties:
    - `processing: false` — true while agent is working
    - `thinkingCollapsed: false` — thinking panel collapse state
    - `previewUrl: null` — image preview ObjectURL
    - `fileName: null` — uploaded file name
    - `messageText: ''` — textarea content

    **CRITICAL:** The `x-data` must be on a stable outer div that HTMX never targets for swap. All `sse-swap` targets are inner leaf elements. Add `@stream-done.window="processing = false"` on this same x-data container to listen for the `stream-done` CustomEvent dispatched by the done/interrupt swap targets (see below).

    **SSE connection on parent wrapper (NOT on chatHistory):**
    Place `hx-ext="sse" sse-connect="/chat/stream"` on a parent wrapper (`id="sseContainer"`) that wraps both the chat column and summary column. Both `hx-ext="sse"` and `sse-connect` MUST be on the same element (Pitfall 1 from research). The chatHistory div becomes a plain child without SSE attributes:
    ```html
    <div id="chatHistory"
         class="flex-1 overflow-y-auto p-6 md:p-8 space-y-8 hide-scrollbar">
    ```

    **SSE swap targets inside chatHistory:**
    - `<div id="tokenTarget" sse-swap="token" hx-swap="beforeend"></div>` — tokens append here during streaming. Use `beforeend` NOT `innerHTML` (Pitfall 3).
    - `<div id="thinkingPanel" sse-swap="thinking-start" hx-swap="innerHTML">` — receives thinking panel open HTML. Contains inner elements:
      - `<div id="thinkingStepName" sse-swap="step-name" hx-swap="innerHTML"></div>` — step label updates
      - `<div id="thinkingContent" sse-swap="step-content" hx-swap="beforeend"></div>` — step entries accumulate
    - `<div id="thinkingDone" sse-swap="thinking-done" hx-swap="innerHTML"></div>` — receives collapsed summary line
    - `<div id="aiMessages" sse-swap="message" hx-swap="beforeend"></div>` — receives final AI message bubble HTML
    - `<div id="doneTarget" sse-swap="done" hx-swap="innerHTML" hx-on::after-swap="document.dispatchEvent(new CustomEvent('stream-done'))"></div>` — done sentinel. When the SSE `done` event fires, HTMX swaps content into this div. The `hx-on::after-swap` dispatches a `stream-done` CustomEvent on the window. On the Alpine x-data container, listen with `@stream-done.window="processing = false"` to reset the processing flag. **Why not `@sse-message`:** HTMX SSE dispatches `htmx:sseMessage` (camelCase with colon), not `sse-message`. The Alpine `@sse-message` listener would never fire.
    - `<div id="interruptTarget" sse-swap="interrupt" hx-swap="innerHTML" hx-on::after-swap="document.dispatchEvent(new CustomEvent('stream-done'))"></div>` — interrupt also resets processing via the same `stream-done` event pattern
    - `<div id="summaryContent" sse-swap="summary-update" hx-swap="innerHTML"></div>` — in the right panel (inside the shared sseContainer wrapper), receives summary HTML

    **Thinking panel UI (CHAT-03, CHAT-09):**
    The thinking panel should be visible when `processing` is true OR when entries exist. Use Alpine `x-show`:
    ```html
    <div x-show="processing || thinkingEntries" class="...">
        <button @click="thinkingCollapsed = !thinkingCollapsed" class="flex items-center gap-2">
            <div class="intelligence-pulse" x-show="processing"></div>
            <span x-show="!processing" id="thinkingDone" sse-swap="thinking-done" hx-swap="innerHTML">Thinking...</span>
            <span class="material-symbols-outlined text-xs transition-transform" :class="thinkingCollapsed ? '' : 'rotate-180'">expand_more</span>
        </button>
        <div x-show="!thinkingCollapsed" x-collapse>
            <div id="thinkingStepName" sse-swap="step-name" hx-swap="innerHTML" class="text-xs text-secondary font-bold"></div>
            <div id="thinkingContent" sse-swap="step-content" hx-swap="beforeend" class="text-xs text-outline space-y-1"></div>
        </div>
    </div>
    ```

    **Chat form (multipart, HTMX POST):**
    Replace the static input area with:
    ```html
    <form id="chatForm"
          hx-post="/chat/message"
          hx-encoding="multipart/form-data"
          hx-swap="none"
          @submit="processing = true; messageText = ''; $nextTick(() => $refs.messageInput.style.height = 'auto')">
        <input type="file" name="receipt" accept="image/*" class="hidden"
               x-ref="fileInput"
               @change="handleFileSelect($event)">
        <div class="flex items-end gap-3 bg-surface-container-lowest border border-outline-variant/30 rounded-[1.25rem] p-2 focus-within:border-secondary/50 transition-all shadow-2xl">
            <button type="button" @click="$refs.fileInput.click()"
                    class="p-3 rounded-xl hover:bg-surface-bright text-outline hover:text-secondary transition-colors">
                <span class="material-symbols-outlined">attach_file</span>
            </button>
            <textarea name="message" x-model="messageText" x-ref="messageInput"
                      class="flex-1 bg-transparent border-none focus:ring-0 text-on-surface placeholder:text-outline/50 py-3 resize-none font-body text-sm"
                      placeholder="Type a message or drop files..." rows="1"
                      @keydown.enter.prevent="if (!$event.shiftKey) $el.form.requestSubmit()"></textarea>
            <button type="submit" :disabled="processing && !messageText.trim()"
                    class="p-3 rounded-xl bg-gradient-to-br from-primary to-primary-dim text-on-primary shadow-lg shadow-primary/20 active:scale-90 transition-transform">
                <span class="material-symbols-outlined">send</span>
            </button>
        </div>
    </form>
    ```

    **Image preview (CHAT-02):**
    Below the form or as an overlay, show the preview when `previewUrl` is set:

    ```html
        <span class="text-xs text-outline" x-text="fileName"></span>
        <button type="button" @click="previewUrl = null; fileName = null; $refs.fileInput.value = ''"
                class="text-outline hover:text-error text-xs">Remove</button>
    </div>
    ```

    Alpine method `handleFileSelect(event)`:
    ```javascript
    handleFileSelect(event) {
        const file = event.target.files[0];
        if (file) {
            this.previewUrl = URL.createObjectURL(file);
            this.fileName = file.name;
        }
    }
    ```

    **Drag-and-drop (CHAT-02):**
    On the chat history area, add Alpine event handlers:
    ```html
    @dragover.prevent="$el.classList.add('ring-2', 'ring-secondary/50')"
    @dragleave="$el.classList.remove('ring-2', 'ring-secondary/50')"
    @drop.prevent="handleDrop($event)"
    ```
    Alpine method `handleDrop(event)`:
    ```javascript
    handleDrop(event) {
        event.target.classList.remove('ring-2', 'ring-secondary/50');
        const file = event.dataTransfer.files[0];
        if (file && file.type.startsWith('image/')) {
            const dt = new DataTransfer();
            dt.items.add(file);
            this.$refs.fileInput.files = dt.files;
            this.previewUrl = URL.createObjectURL(file);
            this.fileName = file.name;
        }
    }
    ```

    **Submission summary panel (right side) — SSE target:**
    Move `hx-ext="sse" sse-connect="/chat/stream"` UP from the chatHistory div to a parent wrapper div that contains BOTH the chat history column AND the summary panel column. This way one single SSE connection serves all swap targets (token, message, thinking-*, done, interrupt, AND summary-update). Do NOT create a second SSE connection — a second EventSource to the same queue-backed endpoint is architecturally broken because the queue is consumed by the first reader.

    Concrete structure:
    ```html
    <div id="sseContainer" hx-ext="sse" sse-connect="/chat/stream">
      <!-- Left column: chat history -->
      <div id="chatHistory" class="flex-1 overflow-y-auto ...">
        <!-- all sse-swap targets for chat: token, message, thinking-*, done, interrupt -->
      </div>
      <!-- Right column: summary panel -->
      <div class="...">
        <div id="summaryContent" sse-swap="summary-update" hx-swap="innerHTML">
          <!-- initial static summary content -->
        </div>
      </div>
    </div>
    ```

    The `x-data="chatState()"` container must be a stable ancestor of this sseContainer div (or sseContainer itself can carry x-data if it is never swapped). The key constraint: sseContainer is never an HTMX swap target.
