<objective>
Close the single blocker gap from 12-UAT.md: the eval suite currently requires ANTHROPIC_API_KEY because `eval/src/capture/runner.py` uses `claude-code-sdk` to spawn a Claude subagent that drives Playwright MCP tools. The user does not have an Anthropic key, and this violates the project's OpenRouter-only convention documented in CLAUDE.md.

Replace the subagent with a direct parameterized `playwright` Python script. The 20 benchmarks are stateless one-shot UI flows (documented in 12-04-SUMMARY.md as `key-decisions`), so a deterministic script is strictly simpler, debuggable, and compliant.

Purpose: Unblock end-to-end evaluation (UAT tests 2-7 all gate on this) without introducing a second LLM provider.

- `eval/src/capture/runner.py` rewritten to use `playwright.async_api` directly.
- `eval/src/capture/resultsStore.py` holding the persistence helpers that `run_eval.py` already consumes (relocated from the obsolete `subagent.py`).
- `eval/src/capture/subagent.py` reduced to nothing or deleted (prompt builders are obsolete).
- `eval/src/config.py` without `anthropicApiKey` / ANTHROPIC_API_KEY validation.
- `pyproject.toml` with `playwright` added to dev group and `claude-code-sdk` removed.
- TDD coverage for the new pure parsing functions and an integration test against a static mock HTML fixture.

Explicitly out of scope (do NOT change):
- `enrichment.py`, `metrics/`, `scoring.py`, `run_eval.py` orchestrator control flow (only its imports need to move from `subagent` to `resultsStore`).
- Judge model wiring (gpt-4o via OpenRouter) or Confident AI push.
- DOM selectors below — these MUST be preserved verbatim (they come from `templates/login.html` and `templates/chat.html`):
  - `input[name="username"]`, `input[name="password"]`, login submit: `button[type="submit"]` on `/login`

    Rewrite `eval/src/capture/runner.py`. Required symbols (public or module-private as noted):

    **Pure functions (no I/O — fully unit-testable):**
    - `_buildErrorResult(benchmark: Benchmark, errorMessage: str) -> dict` — same shape as the old one (already documented in the deleted `runner.py`). Copy the structure verbatim.
    - `parseAgentMessages(messages: list[dict]) -> dict` — takes the conversation transcript list, returns dict with keys `claimId` (str | None), `agentDecision` (str | None), `extractedFields` (dict | None). Implementation:
      * `claimId`: regex `CLAIM-[A-Za-z0-9]+` against concatenated assistant content, return first match or None.
      * `agentDecision`: last assistant message's content, truncated to first 500 chars. If no assistant messages, None.
      * `extractedFields`: scan all assistant content for `Key: Value` lines for keys in `{"merchant", "date", "total", "subtotal", "tax", "currency", "payment"}` (case-insensitive), return dict or None if no keys found. Keep the heuristic simple — enrichment and metrics don't rely on this being perfect; they use the transcript text directly.

    **Playwright functions:**
    - `async def scrapeCapturedDom(page, benchmark: Benchmark) -> dict` — reads `#aiMessages` children, builds `conversationTranscript` (role="assistant" for each child's inner text; user messages are not in the DOM post-stream but the transcript is still consumed by metrics as text context), passes through `parseAgentMessages`, returns the full `capture` dict with enrichment-filled fields set to None / [].
    - `async def _loginSession(page, config: EvalConfig) -> None` — navigates to `{config.appUrl}/login`, fills `input[name="username"]` with `config.evalUsername`, fills `input[name="password"]` with `config.evalPassword`, clicks `button[type="submit"]`, waits for URL to leave `/login` (timeout 10s). Raises `RuntimeError("Login failed")` on timeout.
    - `async def _logoutSession(page, config: EvalConfig) -> None` — navigates to `{config.appUrl}/logout` and waits for redirect to `/login`.
    - `async def _submitReceipt(page, receiptPath: Path, userMessage: str) -> None` — uses `await page.set_input_files('input[type="file"][name="receipt"]', str(receiptPath))`, fills `textarea[name="message"]`, clicks `#chatForm button[type="submit"]`.
    - `async def _waitForStreamOrInterrupt(page, timeoutMs: int = 120000) -> str` — polls in a 500ms loop (or uses `page.wait_for_function`) for `#doneTarget:not(:empty)` OR `#interruptTarget:not(:empty)`. Returns "done" or "interrupt". Raises `TimeoutError` if neither becomes non-empty within the timeout.
    - `async def _handleInterruptIfPresent(page, responseText: str = "Yes, please proceed") -> bool` — if `#interruptTarget` has content, fill the textarea with `responseText`, click submit, clear the interrupt target, return True. Else return False.

    **Single-benchmark capture:**
    - `async def runSingleCapture(benchmark: Benchmark, config: EvalConfig) -> dict`:
      * Validate receipt file exists: `receiptPath = config.invoicesDir / benchmark["file"]`. If not `receiptPath.exists()`, return `_buildErrorResult(benchmark, f"Receipt file not found: {benchmark['file']}")`. This handles ER-018 gracefully.
      * Open Playwright: `async with async_playwright() as p: browser = await p.chromium.launch(headless=True); context = await browser.new_context(); page = await context.new_page()`.
      * For ER-013 (duplicate detection): call `_runDuplicateFlow(page, context, benchmark, receiptPath, config)`. Else: call `_runStandardFlow(page, benchmark, receiptPath, config)`.
      * Wrap in try/except — on any Playwright exception, log and return `_buildErrorResult(benchmark, f"Playwright error: {exc}")`. Always close the browser in a finally block.
      * The flow function returns a `capture` dict; `runSingleCapture` wraps it in the full result schema: `{benchmarkId, benchmark, category, file, scoringType, capture, expected}` — matching the existing schema exactly (see `_buildErrorResult` for the shape and 12-02-SUMMARY.md's captured-output schema).

    - `async def _runStandardFlow(page, benchmark, receiptPath, config) -> dict`:
      1. `await _loginSession(page, config)`
      2. `await _submitReceipt(page, receiptPath, f"Please process this receipt: {benchmark['scenario']}")`
      3. Loop up to 3 times: `status = await _waitForStreamOrInterrupt(page)`. If `status == "interrupt"`, call `_handleInterruptIfPresent(page)` and continue. If `status == "done"`, break.
      4. Return `await scrapeCapturedDom(page, benchmark)`.

    - `async def _runDuplicateFlow(page, context, benchmark, receiptPath, config) -> dict`:
      * SESSION 1: login, submit, wait for done, handle interrupts, note `#aiMessages` state (we don't need to scrape this — just let the first claim land in the DB).
      * Logout: `await _logoutSession(page, config)`.
      * SESSION 2: login again (session cookie cleared by logout), submit the SAME receipt, wait for done, handle interrupts, scrape.
      * Return the Session 2 capture dict — this is what scoring evaluates.

    **Batch runner (preserve existing signature and behavior):**
    - `async def runCapture(benchmarks: list[Benchmark], config: EvalConfig) -> list[dict]`:
      * Iterate sequentially (same as before — avoids session conflicts).
      * For each: log `CAPTURE [idx/total] benchmarkId — benchmark`; call `runSingleCapture`; on exception catch and wrap as `_buildErrorResult(benchmark, f"Unexpected error: {exc}")`; append to results.
      * Print error or agent decision per benchmark, same format as before.
      * `await asyncio.sleep(1)` between benchmarks.
      * Return the results list — same order as input.
