Create `eval/src/capture/runner.py` with the automated subagent invocation loop. This module uses the Anthropic Python SDK to programmatically spawn a Claude agent for each benchmark, providing it with the subagent prompt from `subagent.py` and Playwright MCP tools for browser automation.

**First, add `anthropic` as a dev dependency**: `poetry add anthropic --group dev` (if not already present).

**Core function: `runCapture(benchmarks: list[Benchmark], config: EvalConfig) -> list[dict]`**:

For each benchmark in the list:
1. Build the subagent prompt via `buildCapturePrompt(benchmark, config)` (or `buildDuplicateCapturePrompt` for ER-013)
2. Invoke the Claude API using the `anthropic` Python SDK:

```python
import anthropic

client = anthropic.Anthropic(api_key=config.anthropicApiKey)

# The subagent prompt instructs Claude to use Playwright MCP browser tools
# to interact with the live app and capture structured output.
# Claude uses tool_use to drive Playwright actions.
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=16384,
    system="You are a test automation agent. You have access to Playwright MCP browser tools. Follow the benchmark script exactly. Return ONLY the JSON result object at the end.",
    messages=[
        {"role": "user", "content": capturePrompt}
    ],
    tools=[
        # Playwright MCP tools are provided via the MCP server connection
        # The tool definitions should match what Playwright MCP exposes
    ],
    # NOTE: Tool configuration for Playwright MCP integration.
    # Two approaches to evaluate during implementation:
    #
    # Option A: Use claude-code-sdk (subprocess) which has native MCP support:
    #   from claude_code_sdk import query
    #   result = await query(prompt=capturePrompt, options={"allowedTools": ["mcp__playwright__*"]})
    #
    # Option B: Use anthropic SDK with computer-use / tool-use patterns:
    #   Define Playwright tool schemas explicitly and handle tool_use responses
    #   in a loop (send tool result back, get next action, repeat until done).
    #
    # Prefer Option A (claude-code-sdk) if available as it handles the MCP
    # tool loop natively. Fall back to Option B if claude-code-sdk is not
    # installable or has compatibility issues.
)
```

**Implementation approach -- use `claude-code-sdk`** (preferred):

```python
# poetry add claude-code-sdk --group dev
from claude_code_sdk import query, ClaudeCodeOptions

async def runSingleCapture(benchmark: Benchmark, config: EvalConfig) -> dict:
    """Run a single benchmark capture via Claude subagent with Playwright MCP."""
    if benchmark["benchmarkId"] == "ER-013":
        prompt = buildDuplicateCapturePrompt(benchmark, config)
    else:
        prompt = buildCapturePrompt(benchmark, config)

    resultText = ""
    async for event in query(
        prompt=prompt,
        options=ClaudeCodeOptions(
            allowed_tools=["mcp__playwright__*"],
            max_turns=50,  # enough for login + upload + wait + capture
        ),
    ):
        if event.type == "result":
            resultText = event.text
        elif event.type == "tool_use":
            # Log tool invocations for debugging
            print(f"  [{benchmark['benchmarkId']}] Tool: {event.tool_name}")

    # Parse the JSON result from Claude's final response
    capturedResult = parseSubagentResponse(resultText, benchmark)
    return capturedResult
```

**`parseSubagentResponse(responseText: str, benchmark: Benchmark) -> dict`**:
- Extract JSON from Claude's response text (may be wrapped in markdown code blocks)
- Validate the JSON has the expected schema (benchmarkId, capture, expected)
- If parsing fails, return a partial result with error info:
  ```python
  {
      "benchmarkId": benchmark["benchmarkId"],
      "benchmark": benchmark["benchmark"],
      "category": benchmark["category"],
      "file": benchmark["file"],
      "scoringType": benchmark["scoringType"],
      "capture": {
          "claimId": None,
          "conversationTranscript": [],
          "extractedFields": {},
          "agentDecision": "",
          "complianceFindings": None,
          "fraudFindings": None,
          "advisorReasoning": None,
          "retrievedPolicyChunks": [],
          "captureError": f"Failed to parse subagent response: {str(e)}",
      },
      "expected": {...},
  }
  ```

**`runCapture(benchmarks: list[Benchmark], config: EvalConfig) -> list[dict]`**:
- Iterates over benchmarks sequentially (not parallel -- each needs a clean browser session)
- For each benchmark:
  1. Print progress: `[1/20] Capturing ER-001 (E1_Document_Type_Identification)...`
  2. Call `runSingleCapture(benchmark, config)`
  3. Save result to disk via `saveCapturedResult(result, config.resultsDir)`
  4. Print result: `  -> Captured claim {claimId}, decision: {agentDecision}`
  5. If capture error, print warning and continue to next benchmark
- Returns list of all captured results

**Error handling**:
- If Anthropic API returns an error, catch and record as captureError
- If Claude times out (no result after max_turns), record timeout error
- If Playwright MCP is unreachable, record connection error
- Never crash the loop -- always continue to the next benchmark

CamelCase naming. Async functions. Type hints.

Create `eval/run_eval.py` as the main entry point.

**CLI arguments** (use argparse -- stdlib, no extra deps):
- `--skip-capture`: Skip browser capture phase, load results from `eval/results/` JSON files. For re-scoring without re-running Playwright.
- `--benchmark ER-XXX`: Run a single benchmark instead of all 20. Works with or without `--skip-capture`.
- `--skip-push`: Skip Confident AI dashboard push (useful for local debugging).
- `--verbose`: Print detailed metric reasons for each benchmark.

**Pipeline steps** (run sequentially):

```
1. CAPTURE (skip if --skip-capture)
   Call runCapture(benchmarks, config) from runner.py
   This automatically:
     - Spawns a Claude subagent per benchmark via Anthropic SDK
     - Claude subagent uses Playwright MCP to interact with the live app
     - Captures structured JSON output
     - Saves results to eval/results/{benchmarkId}.json

2. LOAD
   Load all captured results from eval/results/ via loadAllCapturedResults()
   Filter to single benchmark if --benchmark specified

3. ENRICH
   For each captured result:
     enrichCapturedResult(result, config.dbUrl)
   Save enriched results back to eval/results/{benchmarkId}.json

4. SCORE
   For each captured result:
     a. Get benchmark definition from dataset
     b. Get metrics via getMetricsForBenchmark()
     c. Build LLMTestCase via buildTestCase()
     d. Collect into lists for evaluate()

   Call deepeval's evaluate():
     from deepeval import evaluate
     evaluationResult = evaluate(
         test_cases=testCases,
         metrics=allMetrics,
         identifier=f"MMGA-Run-{datetime.now().isoformat()}",
         hyperparameters={
             "judgeModel": "gpt-4o",
             "appVersion": "phase-12",
         },
     )

   NOTE: evaluate() in deepeval 3.x returns EvaluationResult object.
   Access results via evaluationResult.test_results or iterating evaluationResult.

5. REPORT
   Build results list from evaluation output
   Call scoring functions:
     categoryScores = calculateCategoryScores(results)
     overallScore = calculateWeightedOverall(categoryScores)
     primaryTargets = checkPrimaryTargets(results)
     printSummaryTable(results, categoryScores, overallScore)

   Confident AI push happens automatically via evaluate() if CONFIDENT_AI_API_KEY is set.
   If --skip-push, set env var DEEPEVAL_TELEMETRY_OPT_OUT=1 before evaluate().
```

**Error handling:**
- If capture fails for a benchmark, log error and continue (partial results are fine)
- If enrichment fails, continue with partial data (warn in summary)
- If a metric fails to evaluate, record score=0.0 with reason "metric error: {exception}"
- If no captured results found and --skip-capture, exit with error message

**Main function:**
```python
async def main():
    args = parseArgs()
    config = getEvalConfig()
    # ... pipeline steps
```

Entry point:
```python
if __name__ == "__main__":
    import asyncio
    asyncio.run(main())
```

CamelCase naming throughout. Async for capture and enrichment (both are async).
