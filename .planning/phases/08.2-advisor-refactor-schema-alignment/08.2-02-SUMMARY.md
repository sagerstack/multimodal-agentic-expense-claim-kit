## Decisions Made

- Nullable column — existing claims get NULL rather than a forced default. The review page is responsible for display fallback.
- Validation in the tool layer, not the MCP server — the MCP server accepts any string or None; the tool enforces the enum constraint so invalid LLM output is caught before reaching the DB.
- All 3 SQL paths in `insertClaim` updated (idempotent+claimNumber, idempotent+no-claimNumber, non-idempotent) to ensure category is persisted regardless of insertion path.
