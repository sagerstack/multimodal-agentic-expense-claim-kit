<objective>
Add a `category` column to the claims table and wire the full category inference pipeline: the intake agent classifies the expense during extraction (one of meals, transport, accommodation, office_supplies, general), passes it through submitClaim to the DB. This fixes the bug where category always shows "General" because it was being incorrectly extracted from line_items JSON.

Purpose: Category is a first-class claim attribute that drives policy search and review page display. Currently it's hardcoded to "General" because no column exists and the code tries to read it from line_items (which contains receipt items, not a category).

Output: Migration, ORM update, system prompt instruction, submitClaim wiring, MCP DB server parameter.
</objective>
