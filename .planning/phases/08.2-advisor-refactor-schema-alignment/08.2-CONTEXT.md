# Phase 8.2: Advisor Agent Refactor + Schema Alignment - Context

**Gathered:** 2026-04-07
**Status:** Ready for planning

<domain>
## Phase Boundary

Improve data quality and display accuracy across the Review page. Capture real advisor LLM reasoning, enforce consistent intakeFindings schema, fix category inference, enhance the Review page cards, and add two new pages (Manage, Analytics). The advisor remains a ReAct agent (no architecture refactor).

</domain>

<decisions>
## Implementation Decisions

### Advisor reasoning capture
- Store the full LLM response text (final AIMessage content) in `advisor_findings` JSONB column, as-is
- No structured extraction or parsing of the LLM text — store raw
- Display as plain text in the Advisory Agent card, no markdown rendering
- Fixed card height with scrollbar for overflow — do not truncate
- Submission history (employee claim count + approval rate) stays in the Advisory Agent card, below the reasoning text
- When `advisor_findings` is NULL, show "Awaiting advisor analysis." pending message — no audit_log fallback
- Rename `aiInsight` variable to `submissionHistory` across router and templates

### Review page cards
- Per-field confidence scores shown as mini progress bars: field name | thin colored bar | percentage
  - Green >= 90%, yellow 75-89%, red < 75%
- "Extracted Entity Data" renamed to "Extracted Claim Information"
- Sidebar shows: receipt attributes (line items, receipt number) + claim metadata (claim number, submission date, status)
- Old review page (`review.html`) removed entirely — not archived
- Review v2 becomes the default at `/review/{claimId}`

### Category inference
- New `category` VARCHAR column on claims table (migration)
- Uses the 5 policy document categories exactly: meals, transport, accommodation, office_supplies, general
- Inferred during Phase 1 (extraction) from receipt content (merchant name, line items)
- Category drives the policy search query in Phase 2
- System prompt instruction added for the intake agent to classify
- Ambiguous receipts: default to best guess, flag as low confidence in intakeFindings confidenceScores
- Category is NOT added to the mandatory intakeFindings schema — it lives on the claims table column only

### New pages
- **Manage page** (`/manage`): reviewer-only, no Stitch design files available — design following existing dark theme (dashboard.html aesthetic)
  - Filters: status, date range, category (no employee filter)
  - Bulk actions on claims: approve/reject multiple, filter by status/date/category
- **Analytics page** (`/analytics`): reviewer-only, no Stitch design files available — design following existing dark theme (dashboard.html aesthetic)
  - Real-time SQL aggregations on page load — KPIs, charts, and tables reflect current DB state
- Both pages appear in sidebar nav for reviewer role only (same visibility as existing Review link)

### Claude's Discretion
- Exact mini progress bar styling and colors
- Card spacing and typography adjustments
- SQL query structure for analytics aggregations
- Manage page table column selection and sort defaults

</decisions>
