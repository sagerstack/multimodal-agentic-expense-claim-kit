    **1. System prompt** (`prompts/fraudSystemPrompt.py`):
    Port `FRAUD_SYSTEM_PROMPT` verbatim from reference. Covers detection rules: duplicate (high confidence), date proximity (medium), frequency anomaly (medium), amount anomaly (low), legit (default).

    **2. Query helpers** (`tools/queryClaimsHistory.py`) — CRITICAL SQL INJECTION FIX:
    The reference code uses f-string interpolation for SQL queries (e.g. `WHERE c.employee_id = '{employeeId}'`). The DB MCP `executeQuery` tool only accepts a raw SQL string (no parameter binding). Fix by sanitizing inputs:
    - Create a `_sanitize(value: str) -> str` helper that escapes single quotes by replacing `'` with `''` (standard SQL escaping). This is the minimal viable fix since executeQuery doesn't support parameterized queries.
    - Apply `_sanitize()` to all string values interpolated into SQL: employeeId, merchant, receiptDate.
    - For numeric values (amountSgd, days), cast to float/int before interpolation to prevent injection.
    - Port all 3 query functions: `exactDuplicateCheck`, `recentClaimsByEmployee`, `claimsByMerchantAndEmployee`.
    - Each function: build SQL with sanitized values, call `mcpCallTool` with `executeQuery`, handle errors gracefully (return empty list).

    **3. Node** (`node.py`):
    - Use `buildAgentLlm` from `agents.shared.llmFactory` instead of local factory.
    - Use `extractJsonBlock` from `agents.shared.utils` instead of local duplicate.
    - Keep `_parseFraudResponse`, `_isExactDuplicate`, `_computeAverage`, `_countMerchantIn30Days` as local helpers (fraud-specific logic).
    - Node workflow:
      a. Read extractedReceipt, intakeFindings from state
      b. Extract employeeId, merchant, receiptDate, totalAmountSgd from fields
      c. Run 3 DB queries via query helpers (each wrapped in try/except)
      d. Check `_isExactDuplicate` — if true, return deterministic duplicate verdict immediately (NO LLM call)
      e. If not duplicate: compute statistics (average, frequency), build LLM context, call LLM with 402 fallback
      f. Parse response with `_parseFraudResponse`
      g. Write audit_log entry: `insertAuditLog` with action="fraud_check", newValue=JSON of fraudFindings (verdict + flags + summary), actor="fraud_agent", claimId from `state.get("dbClaimId")`
      h. Return `{"messages": [AIMessage(summaryMsg)], "fraudFindings": fraudFindings}`

    4. Create `__init__.py` files for prompts/ and tools/ packages.
