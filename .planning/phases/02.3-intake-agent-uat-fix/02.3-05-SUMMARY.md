## Decisions Made

**ConversationRunner architecture:**
- Uses ainvoke() instead of astream_events() for reliability (synchronous execution, simpler state management)
- Image upload via storeImage() with file path (same pattern as app.py but from filesystem)
- TurnResult captures messages (list[str]), steps (list[StepRecord]), interrupt state (bool), interrupt question/data
- Interrupt detection via aget_state() checking finalState.tasks for interrupts (same as app.py)
- Message count tracking for deduplication between turns

**E2E test design:**
- Structural assertions handle LLM non-determinism (tool names, CLAIM-NNN regex, concept keywords like "dig", "sgd", "policy")
- Tool ordering validation ensures narrative flow: extractReceiptFields -> convertCurrency -> searchPolicies -> submitClaim
- Handles both interrupt (askHuman) and non-interrupt agent behavior (robust to prompt variations)
- Separate narration test validates prompt compliance (agent speaks before tools)
- DIG restaurant receipt ($16.20 USD) validates end-to-end flow with real artifact

**.env.e2e configuration:**
- localhost:8001-8004 MCP URLs match Docker Compose port mappings (mcp-rag, mcp-db, mcp-currency, mcp-email)
- LOG_LEVEL=INFO reduces noise, SEQ_INGESTION_URL empty (no Docker-internal Seq)
- APP_ENV=e2e for environment tracking
- Real OpenRouter API key required (not mocked)

**Interactive CLI:**
- upload:path/to/file.jpg syntax for image input
- python -m agentic_claims.cli for terminal-based chat
- Uses .env.e2e by default (can override with envFile param)
