LangGraph astream_events API:
- `graph.astream_events(input, config, version="v2")` yields events as they happen
- Event kinds relevant: "on_chat_model_start", "on_chat_model_stream", "on_tool_start", "on_tool_end"
- Each event has: event (kind string), name, data, metadata
- For tool events: data contains tool name and args/output
- For chat events: data contains content chunks

Key constraint: The current code uses `graph.ainvoke()` which returns the final state. With `astream_events()`, we need to:
1. Accumulate the final state from events (or call `aget_state` after streaming completes)
2. Stream CoT into the Step panel as events arrive
3. Still extract user-facing messages vs CoT entries from the final state

  **Streaming refactor:**

  Replace the graph.ainvoke() call with astream_events(). The new flow:

  ```python
  import time

  async with cl.Step(name="Thinking", type="tool") as processingStep:
      startTime = time.time()
      processingStep.input = (
          "Extracting receipt fields, validating policy, converting currency" if imageB64
          else "Handling user response"
      )

      cotEntries = []

      if awaitingClarification:
          invokeInput = Command(resume={...})
      else:
          invokeInput = inputState

      # Stream events for real-time CoT
      async for event in graph.astream_events(
          invokeInput,
          config={"configurable": {"thread_id": threadId}},
          version="v2",
      ):
          kind = event.get("event", "")

          # Capture tool calls as CoT
          if kind == "on_tool_start":
              toolName = event.get("name", "unknown")
              toolInput = str(event.get("data", {}).get("input", ""))[:200]
              entry = f"**Tool Call:** {toolName}({toolInput})"
              cotEntries.append(entry)
              processingStep.output = "\n\n".join(cotEntries)
              await processingStep.update()

          # Capture tool results as CoT
          elif kind == "on_tool_end":
              toolName = event.get("name", "unknown")
              toolOutput = str(event.get("data", {}).get("output", ""))[:500]
              entry = f"**Tool Result ({toolName}):** {toolOutput}"
              cotEntries.append(entry)
              processingStep.output = "\n\n".join(cotEntries)
              await processingStep.update()

      # After streaming completes, update step name with duration
      elapsed = time.time() - startTime
      if elapsed >= 60:
          minutes = int(elapsed // 60)
          seconds = int(elapsed % 60)
          processingStep.name = f"Thought for {minutes} min {seconds} seconds"
      else:
          processingStep.name = f"Thought for {int(elapsed)} seconds"

      if not cotEntries:
          processingStep.output = "No intermediate reasoning captured"
  ```

  **After streaming: get final state for message extraction**

  After the streaming loop, get the current state to extract user-facing messages:

  ```python
  # Get final state after streaming
  finalState = await graph.aget_state(config={"configurable": {"thread_id": threadId}})
  result = finalState.values
  ```

  Then use the existing message extraction logic (messageCountBefore, userFacingMessages, etc.) but operating on `result` from `aget_state` instead of the ainvoke return value.

  **Handle interrupts:**

  The interrupt detection needs adjustment. With astream_events, check the final state for pending interrupts:
  ```python
  # Check for interrupts in the final state
  if finalState.next:  # If there are pending nodes, likely an interrupt

      ...
  ```

  Actually, a simpler approach: check `finalState.tasks` or inspect the state values for `__interrupt__` key. If the graph is interrupted, `finalState.next` will be non-empty (the interrupted node name).

  For interrupt handling, use:
  ```python
  # Check for interrupt
  if finalState.next:
      # Graph is paused (interrupt) -- check for askHuman interrupt payload
      for task in finalState.tasks:
          if hasattr(task, 'interrupts') and task.interrupts:
              interruptPayload = task.interrupts[0].value
              question = interruptPayload.get("question", "Please confirm or correct:")
              data = interruptPayload.get("data", {})
              await cl.Message(content=question).send()
              if data:
                  await cl.Message(content=f"```json\n{json.dumps(data, indent=2)}\n```").send()
              cl.user_session.set("awaiting_clarification", True)
              return
  ```
