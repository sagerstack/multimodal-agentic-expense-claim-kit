**Architecture change — batch-then-display to streaming-and-interleave:**

```
CURRENT (batch):
  graph.ainvoke()  ──────────────────────────────►  result
  │  One cl.Step("Processing Receipt") wraps       │  Extract messages
  │  the entire invoke. CoT batched at end.        │  from result, send
  │  User sees empty Step, then all messages.      │  all at once.
  └────────────────────────────────────────────────┘

NEW (streaming):
  graph.astream_events(version="v2")  ──►  event  ──►  event  ──►  ...
                                            │           │
                                            ▼           ▼
                                     on_chat_model_stream  on_tool_start
                                     → stream tokens       → open Step("Thinking")
                                       to cl.Message       on_tool_end
                                                           → close Step("Thought for X")
```

**LangGraph astream_events v2 event structure (from research):**

| Event | `event["name"]` | `event["data"]` |
|-------|-----------------|-----------------|
| on_chat_model_stream | model name | {"chunk": AIMessageChunk(content="...", tool_call_chunks=[...])} |
| on_chat_model_end | model name | full AIMessage |
| on_tool_start | tool name (e.g. "extractReceiptFields") | tool input args |
| on_tool_end | tool name | tool output |

AIMessageChunk.content = user-facing text (stream to chat)
AIMessageChunk.tool_call_chunks = tool call args (ignore for display)

**Chainlit API (from research):**

- Message streaming: `msg = cl.Message(content=""); await msg.send(); await msg.stream_token(token); await msg.update()`
- Manual Step: `step = cl.Step(name="Thinking", type="tool"); await step.send(); ... step.name = "Thought for X"; await step.update()`
- Multiple sequential Steps work correctly -- each shows as separate collapsible element

**Known risk:** LangGraph Issues #3071, #5249 report problems with event propagation from nested ReAct agents. If inner ReAct agent events don't bubble up, the streaming loop will complete without capturing tool events. Mitigation: after streaming, use `aget_state()` to get final state and fall back to the old message extraction if no events were captured.

**Event filtering:** Use `event["name"]` to identify tools. Filter out chain/graph infrastructure events (on_chain_start, etc.) -- only process on_chat_model_stream, on_chat_model_end, on_tool_start, on_tool_end.

  **New streaming architecture:**

  After building `invokeInput` (either Command(resume=...) or inputState), replace the graph.ainvoke() call with:

  ```python
  # State tracking for streaming
  currentStep = None      # Currently open Step element (for tool calls)
  stepStartTime = None    # When current Step opened
  currentMsg = None       # Currently streaming cl.Message
  eventsReceived = False  # Track if we got any tool/chat events (fallback detection)

  config = {"configurable": {"thread_id": threadId}}

  async for event in graph.astream_events(
      invokeInput,
      config=config,
      version="v2",
  ):
      kind = event.get("event", "")

      # ── LLM streaming content (user-facing text) ──
      if kind == "on_chat_model_stream":
          chunk = event.get("data", {}).get("chunk")
          if chunk and hasattr(chunk, "content") and chunk.content:
              # Only stream if this is actual content (not tool_call_chunks)
              if not (hasattr(chunk, "tool_call_chunks") and chunk.tool_call_chunks and not chunk.content):
                  if currentMsg is None:
                      currentMsg = cl.Message(content="")
                      await currentMsg.send()
                  await currentMsg.stream_token(chunk.content)
                  eventsReceived = True

      # ── LLM finished a response ──
      elif kind == "on_chat_model_end":
          if currentMsg and currentMsg.content:
              await currentMsg.update()  # Finalize the streamed message
              currentMsg = None

      # ── Tool starting ──
      elif kind == "on_tool_start":
          # Finalize any pending message before opening Step
          if currentMsg and currentMsg.content:
              await currentMsg.update()
              currentMsg = None

          toolName = event.get("name", "unknown")
          currentStep = cl.Step(name="Thinking", type="tool")
          currentStep.input = toolName
          await currentStep.send()
          stepStartTime = time.time()
          eventsReceived = True

          logger.info(f"Tool started: {toolName}")

      # ── Tool finished ──
      elif kind == "on_tool_end":
          if currentStep:
              toolName = event.get("name", "unknown")
              toolOutput = str(event.get("data", {}).get("output", ""))[:500]
              currentStep.output = f"**{toolName}:** {toolOutput}"

              elapsed = time.time() - stepStartTime
              if elapsed >= 60:
                  minutes = int(elapsed // 60)
                  seconds = int(elapsed % 60)
                  currentStep.name = f"Thought for {minutes} min {seconds} seconds"
              else:
                  currentStep.name = f"Thought for {int(elapsed)} seconds"

              await currentStep.update()
              currentStep = None
              stepStartTime = None

              logger.info(f"Tool completed: {toolName}, elapsed: {elapsed:.1f}s")

  # ── Finalize any remaining open elements ──
  if currentMsg and currentMsg.content:
      await currentMsg.update()
  if currentStep:
      elapsed = time.time() - stepStartTime if stepStartTime else 0
      currentStep.name = f"Thought for {int(elapsed)} seconds"
      await currentStep.update()
  ```

  **After streaming: interrupt detection and fallback**

  After the streaming loop completes:

  ```python
  # Get final state for interrupt detection
  finalState = await graph.aget_state(config=config)

  # Check for interrupt (askHuman)
  if finalState.next:
      for task in finalState.tasks:
          if hasattr(task, 'interrupts') and task.interrupts:
              interruptPayload = task.interrupts[0].value
              question = interruptPayload.get("question", "Please confirm or correct:")
              data = interruptPayload.get("data", {})
              await cl.Message(content=question).send()
              if data:
                  await cl.Message(content=f"```json\n{json.dumps(data, indent=2)}\n```").send()
              cl.user_session.set("awaiting_clarification", True)
              return

  # FALLBACK: If no streaming events were captured (nested graph issue),
  # fall back to extracting messages from final state
  if not eventsReceived:
      logger.warning("No streaming events captured -- falling back to state extraction")
      result = finalState.values
      if isinstance(result, dict) and "messages" in result:
          # Get messageCountBefore from session or use 0
          for msg in result["messages"]:
              if hasattr(msg, "type") and msg.type == "ai" and hasattr(msg, "content") and msg.content:
                  await cl.Message(content=msg.content).send()
  ```
