## Objective Achieved

Fixed four critical prompt-level issues that made the agent appear broken despite working tools: invisible policy check results (Issue C), bracket-notation placeholders in currency conversion and summary (Issues D, E), lack of error self-diagnosis (Issue B), and field name misalignment with submitClaim tool. Added narration pattern for transparency and claim ID pre-generation.

## Changes Made

### Files Modified

**src/agentic_claims/agents/intake/prompts/agentSystemPrompt.py**
- Added NARRATION PATTERN section with mandatory narration before all 4 tools (extractReceiptFields, convertCurrency, searchPolicies, submitClaim)
- Removed all bracket-notation placeholders ([claimantId], [amount], [rate]) from template sections
- Added mandatory visible policy check message in Step 9 (shows PASS or VIOLATION with cited policy clauses)
- Fixed Step 5 currency conversion ordering: narrate → tool → result (never output placeholders)
- Aligned Step 4 field names with submitClaim interface (employeeId, totalAmount, not claimantId/amountSgd)
- Added Step 10 claim ID pre-generation (CLAIM-NNN format, 3-digit zero-padded)
- Updated Step 11 to pass pre-generated claimNumber and correct field names
- Added ERROR HANDLING section with self-diagnosis protocol (read error, diagnose, self-correct, or explain in user terms)
- Updated few-shot Example 1 to show visible policy check message and claim number in summary
- Added three new constraints: narrate before every tool, no placeholders, self-diagnose errors

## Decisions Made

| Decision | Rationale | Impact |
|----------|-----------|--------|
| Narration pattern before ALL tool calls | Creates transparency, user sees agent's intent before "Thinking" spinner | Works with Plan 04 streaming architecture |
| No bracket-notation placeholders anywhere | Agent waits for tool results before presenting data, prevents broken UX like "Converting [amount] to SGD at rate [rate]" | Cleaner user experience |
| Mandatory visible policy check message | User MUST see which policies were checked and result BEFORE summary card | Resolves Issue C (invisible policy checks) |
| Claim ID pre-generation in Step 10 | User sees claim number in summary BEFORE confirming, then passed to submitClaim | Aligns with common user expectation |
| Self-diagnosis protocol in ERROR HANDLING | Agent reads error, attempts one self-correction, then explains in user terms | Resolves Issue B (agent not handling tool errors) |

## Issues Resolved

- **Issue D**: Currency conversion placeholder text → Fixed Step 5 ordering (narrate → tool → result)
