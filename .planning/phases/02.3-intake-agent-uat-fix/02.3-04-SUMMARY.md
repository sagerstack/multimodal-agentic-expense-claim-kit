## What Was Done

### Task 1: Rewrite onMessage with Streaming Event Loop

**Previous Architecture (Batch-then-Display):**
- Single `cl.Step("Processing Receipt")` wrapped entire graph.ainvoke()
- User saw empty Step panel while agent processed
- All messages dumped at the end
- CoT entries collected in batch, written to Step.output after completion

**New Architecture (Streaming-and-Interleave):**
- `graph.astream_events()` replaces `graph.ainvoke()`
- Event handlers:
  - **on_chat_model_stream:** Stream tokens to `cl.Message.stream_token()` in real-time
  - **on_chat_model_end:** Finalize streamed message via `update()`
  - **on_tool_start:** Finalize pending message, open `cl.Step("Thinking")`
  - **on_tool_end:** Close Step with `"Thought for X seconds"` (or `"X min Y seconds"`)
- Multiple sequential Step elements appear (one per tool call)
- Messages and Steps interleave naturally

**Interrupt Handling:**
- `astream_events()` doesn't expose `__interrupt__` in result
- Use `aget_state()` after streaming completes
- Check `finalState.next` (empty) and `finalState.tasks` for interrupts

**Fallback:**
- If `eventCount == 0` (nested graph issue), extract last AI message from state
- Prevents silent failure when streaming doesn't emit events

**Changes:**
- Added `import time` for elapsed time calculation
- Removed: single Step wrapper, cotEntries/userFacingMessages batch logic, messageCountBefore tracking
- Updated welcome message: `"Welcome! I'm your expense claims assistant. Please upload a receipt image to get started."`

- Differentiates Steps from main chat messages visually

## Decisions Made

| # | Decision | Rationale | Alternatives Considered | Date |
|---|----------|-----------|------------------------|------|
| 1 | Use `astream_events(version="v2")` | LangGraph's streaming API for event-based tool/LLM hooks | `astream()` (no event hooks), `ainvoke()` (no streaming) | 2026-03-26 |
| 2 | Interrupt detection via `aget_state()` after streaming | `astream_events()` doesn't expose `__interrupt__` in result | Modify graph to emit custom event (too complex) | 2026-03-26 |
| 3 | Finalize pending message before opening Step | Ensures clean message/Step boundary | Stream tokens into Step (wrong UX, Steps are for tool CoT) | 2026-03-26 |
| 4 | Format elapsed time as "X min Y seconds" or "X seconds" | Natural language, more readable than "97s" | Raw seconds, milliseconds (too technical) | 2026-03-26 |
| 5 | CSS selectors for Step elements | Target `.step-container`, `.step-header`, `.step-content` | Inline styles via `cl.Step(style={...})` (not supported) | 2026-03-26 |
| 6 | Fallback for nested graph (eventCount == 0) | Nested graphs don't emit events to top-level stream | Fail loudly, skip output (silent failure worse than stale message) | 2026-03-26 |

## Issues Resolved

### Issue C: Empty Step Panel During Processing (UAT-2 Issue C)
**Problem:** User saw empty "Processing Receipt" Step while agent processed, then all messages appeared at end
**Root cause:** Single Step wrapped entire graph.ainvoke() with batch message extraction
**Fix:** Streaming event loop creates per-tool-call Steps in real-time
**Verification:** Step elements appear as tools execute, showing "Thinking" → "Thought for X seconds"

### Issue D: No Real-Time Feedback (UAT-2 Issue D)
**Problem:** User had no indication of progress during agent processing
**Root cause:** Batch-then-display architecture delayed all output until completion
**Fix:** LLM tokens stream to main chat via `stream_token()` as they arrive
**Verification:** User sees narration appear token-by-token in real-time

### Issue E: Unnatural Conversational Rhythm (UAT-2 Issue E)
**Problem:** All messages dumped at once, no interleaving with tool execution
**Root cause:** Single Step wrapper, batch message extraction after invoke
**Fix:** Messages and Steps interleave naturally (narration → Thinking → result → next narration → next Thinking)
**Verification:** Conversation flows like human thought process

## Lessons Learned

### What Went Well
- `astream_events(version="v2")` provided clean hooks for tool/LLM events
- `stream_token()` API made real-time streaming straightforward
- Custom CSS integration worked without Chainlit code changes

### What Could Be Improved
- Nested graph streaming: LangGraph doesn't emit events for nested graphs to top-level stream (fallback required)
- Interrupt detection: `astream_events()` doesn't expose `__interrupt__` (required `aget_state()` workaround)

### Technical Insights
- Finalize pending messages before opening Steps (ensures clean boundaries)
- Track tool start time for accurate elapsed time calculation
- CSS selectors for Step elements: `.step-container`, `.step-header`, `.step-content`
- Fallback pattern: if `eventCount == 0`, extract from state (nested graph issue)
