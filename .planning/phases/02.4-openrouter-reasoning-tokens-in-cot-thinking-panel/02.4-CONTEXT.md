# Phase 2.4: CoT Thinking Panel + Bug Fixes - Context

**Gathered:** 2026-03-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix 5 open bugs from Phase 2.3 UAT (SSL certs, thinking panel visibility, claim number collisions, duplicate submissions, finalResponse fragility) and add reasoning token support to the CoT thinking panel. BUG-010 (post-submission stubs produce no output) is excluded -- that's a Phase 3 concern when real agents replace stubs.

Bug reference: `docs/project_notes/bugs.md`

</domain>

<decisions>
## Implementation Decisions

### Thinking Panel Appearance (BUG-007)
- Collapsed by default (user clicks to expand)
- Must work in both light and dark Chainlit themes -- auto-detect via CSS `prefers-color-scheme` or Chainlit theme variables
- Panel positioned above the agent's response message (current position, keep it)
- Subtle container styling: light border, slight background tint, not visually heavy
- Current CSS uses `rgba(255,255,255,...)` which is invisible in light mode -- replace with theme-aware colors

### Thinking Panel Content
- Each tool entry shows: tool name + elapsed time + one-line summary of what it did
- Summary line in collapsed state: "Thought for 5s · 1,240 tokens · 3 tools" (combines thinking time, token count, tool count)
- Reasoning text (when available) interleaved chronologically with tool calls -- shows the agent's thought process as it unfolds
- When model doesn't support reasoning tokens (e.g., Gemini Flash), panel shows tools only -- no empty "Reasoning" section, no error
- Reasoning content displayed inside a max-height container (e.g., 300px) with internal scrollbar for long reasoning blocks
- Timestamps on every message in the chat (both user and agent messages) -- relative format ("2s ago", "1m ago")

### Claim Number Strategy (BUG-008, BUG-009)
- Format: CLAIM-NNN with DB sequence (auto-incrementing, e.g., `nextval('claim_number_seq')`)
- DB generates claim number on insert -- agent does NOT generate or guess the number
- User sees claim number AFTER successful submission (in the success message), not in the pre-submission summary card
- System prompt updated: agent no longer generates CLAIM-NNN in Phase 2, receives it from submitClaim response in Phase 3
- On startup (dev mode), truncate claims and receipts tables for clean test state -- avoids collision issues during development

### Duplicate Submission Handling (BUG-008)
- Idempotency key per submission: derive from claimId + threadId (or similar unique combination)
- DB upsert on conflict: if idempotency key already exists, return the existing claim instead of inserting a duplicate
- Agent never sees a submission failure from duplicates -- the MCP tool handles dedup transparently
- Agent auto-retries on genuine failures (network timeout, DB error) using existing ERROR HANDLING protocol -- idempotency key ensures retries are safe
- Claims are immutable once submitted -- no modifications allowed after submission
- User can cancel a submitted claim and start a new one (cancel sets status to 'cancelled', new conversation creates fresh claim)

### Reasoning Token Display
- OpenRouter `reasoning` parameter added to model configuration (enabled when model supports it)
- `reasoning_details` extracted from OpenRouter responses during streaming
- Reasoning text interleaved chronologically with tool calls in the thinking panel
- When model doesn't return reasoning tokens, panel gracefully falls back to tool-call-only display
- No separate "Reasoning" section -- just chronological interleaving

### SSL Certificate Fix (BUG-006)
- Add `ca-certificates` package to Dockerfile
- Ensure `certifi` Python package is up to date
- Verify TLS connectivity to OpenRouter from inside the container

### finalResponse Buffer Fix (BUG-011)
- Only assign `finalResponse` from the last `on_chat_model_end` event that has no tool calls
- Intermediate LLM generations between tool calls are discarded from finalResponse assignment
- Track whether all tool calls have completed before treating an `on_chat_model_end` as the final response

### Chat Metrics
- Token count and thinking time displayed in the thinking panel summary line (not as a separate UI element)
- Format: "Thought for 5s · 1,240 tokens · 3 tools"

### Claude's Discretion
- Exact CSS color values for light/dark theme panel styling
- DB sequence naming and reset strategy details
- Idempotency key hashing/generation approach
- How to detect whether current model supports reasoning tokens (config flag vs runtime detection)
- Internal scrollbar styling for reasoning content overflow

</decisions>

<specifics>
## Specific Ideas

- Thinking panel should feel informational, not noisy -- collapsed by default means users who don't care never see it
- Timestamps use relative format like Slack ("2s ago") not absolute clocks
- DB-generated claim numbers eliminate an entire class of LLM-generated data bugs
- Startup truncation is dev-mode only -- controlled by env var or startup script, not application logic

</specifics>

<deferred>
## Deferred Ideas

- Post-submission agent visibility (BUG-010) -- Phase 3 when compliance/fraud/advisor are real LLM agents
- Cumulative session metrics (total tokens, total time) in a persistent status bar -- revisit in Phase 5 (demo polish)
- Live-updating relative timestamps (auto-refresh "2s ago" → "1m ago") -- nice-to-have, not required

</deferred>
