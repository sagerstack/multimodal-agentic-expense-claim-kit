**Note on deferred features:** Per 02.4-RESEARCH.md "Primary recommendation" and "Open Questions #1", reasoning token capture/display (criteria 6-9) was intentionally deferred. Research found that `ChatOpenAI` with `base_url` override does not preserve non-standard response fields like `reasoning_details` from OpenRouter. Implementing this feature requires migrating from `langchain-openai.ChatOpenAI` to `langchain-openrouter.ChatOpenRouter`. The research recommendation was: "Fix the 5 bugs in Phase 2.4 using current stack. Schedule reasoning tokens as a separate phase when team confirms the requirement." This is not a gap — it's a deliberate architectural decision to defer until migration timing is decided.
