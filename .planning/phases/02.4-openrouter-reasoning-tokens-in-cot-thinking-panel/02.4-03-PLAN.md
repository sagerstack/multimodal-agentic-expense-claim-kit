  **BUG-011 Fix: Robust finalResponse detection**

  The current `on_chat_model_end` handler assigns `finalResponse = tokenBuffer` every time it sees an event without tool_calls. In a ReAct loop, intermediate LLM generations (narration between tool calls) can trigger this, causing `finalResponse` to be set to intermediate text. It works now only because the actual final response overwrites it, but this is fragile.

  Fix: Instead of assigning `finalResponse` directly, always overwrite it. The current code actually does this correctly (each non-tool on_chat_model_end overwrites the previous), but we can make it more robust by:

  1. Adding a `pendingToolCalls` counter. Increment on `on_tool_start`, decrement on `on_tool_end`. This tracks whether we're inside a tool execution sequence.

  2. In the `on_chat_model_end` handler:
     - If `hasToolCalls` is True: clear tokenBuffer (intermediate narration — about to call tools)
     - If `hasToolCalls` is False AND `pendingToolCalls == 0`: this is a candidate for finalResponse
     - Always overwrite `finalResponse` with the latest candidate (the last one wins, which is the actual final response)

  The key change: add `pendingToolCalls = 0` to the buffer variables block. Increment in `on_tool_start`, decrement in `on_tool_end`. Add a log line in `on_chat_model_end` that includes `pendingToolCalls` count for debugging.

  Updated `on_chat_model_end` handler:
  ```python
  elif eventKind == "on_chat_model_end":
      output = event.get("data", {}).get("output")
      hasToolCalls = (
          output
          and hasattr(output, "tool_calls")
          and output.tool_calls
      )

      if hasToolCalls:
          # About to call tools — discard narration tokens
          tokenBuffer = ""
      else:
          # Candidate final response — overwrite (last one wins)
          finalResponse = tokenBuffer
          tokenBuffer = ""

      logger.info(
          f"LLM generation ended - intermediate: {hasToolCalls}, "
          f"pending_tools: {pendingToolCalls}, "
          f"buffer_length: {len(finalResponse or tokenBuffer)}"
      )
  ```

  Updated tool handlers:

      currentToolName = event.get("name", "unknown")
      currentToolStartTime = time.time()
      pendingToolCalls += 1
      logger.info(f"Tool call started - name: {currentToolName}, pending: {pendingToolCalls}")
