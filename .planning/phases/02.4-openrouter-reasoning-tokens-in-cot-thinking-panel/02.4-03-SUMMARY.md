# Phase 02.4 Plan 03: Robust finalResponse Detection + Thinking Panel Metrics Summary

Fixed BUG-011 (premature finalResponse assignment) by adding pendingToolCalls counter and enhanced thinking panel summary with tool count display.

## One-liner

Added pendingToolCalls counter to track active tool executions, fixed finalResponse detection to use last-wins strategy, and enhanced thinking panel summary line to show "Thought for 5s · 3 tools".

## What Was Delivered

### BUG-011 Fix: Robust finalResponse Detection

**Problem**: The current `on_chat_model_end` handler assigned `finalResponse = tokenBuffer` every time it saw an event without tool_calls. In a ReAct loop, intermediate LLM generations (narration between tool calls) would trigger this assignment, causing `finalResponse` to be set to intermediate text. It worked only because the actual final response overwrote it, but this was fragile and would break if the LLM stopped after an intermediate generation.

**Solution**:
- Added `pendingToolCalls = 0` counter to track active tool executions
- Increment on `on_tool_start`, decrement on `on_tool_end`
- Enhanced log messages to include `pending_tools: N` for debugging
- Changed assignment logic to always overwrite (last wins strategy)
- Added clarifying comments: "About to call tools" vs "Candidate final response"

**Result**: finalResponse now robustly captures only the actual final LLM response (the last `on_chat_model_end` event with no tool_calls). Intermediate generations are safely discarded.

### Thinking Panel Summary Enhancement

**Added**: Tool count display in the summary line of the thinking panel.

**Format**: `"Thought for 5s · 3 tools"` (or "1 tool" for singular)

**Implementation**:
```python
toolCount = len(toolCalls)
toolLabel = "tool" if toolCount == 1 else "tools"
f'<summary>Thought for {elapsed} · {toolCount} {toolLabel}</summary>\n'
```

**Future-ready**: The format prepares for reasoning token display when available: `"Thought for 5s · 1,240 tokens · 3 tools"`. The infrastructure is ready — when reasoning tokens are exposed by ChatOpenAI, add them between elapsed time and tool count.

### Enhanced Debug Logging

Added `pending_tools` to log messages:
- `on_chat_model_end`: Shows pending_tools count alongside intermediate/buffer_length
- `on_tool_start`: Shows pending count after increment
- `on_tool_end`: Shows pending count after decrement

Example log output:
```
LLM generation ended - intermediate: True, pending_tools: 2, buffer_length: 0
Tool call started - name: convertCurrency, pending: 3
Tool call completed - name: extractReceiptFields, elapsed: 2.45s, pending: 2
```
