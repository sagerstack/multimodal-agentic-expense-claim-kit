    Create src/agentic_claims/core/graph.py:
    - Import StateGraph, START, END from langgraph.graph
    - Import AsyncPostgresSaver from langgraph.checkpoint.postgres.aio
    - Import ClaimState from agentic_claims.core.state
    - Import all 4 node functions from their agent modules
    - Import getSettings from agentic_claims.core.config

    Define `def buildGraph() -> StateGraph`:
    - Create `builder = StateGraph(ClaimState)`
    - Add 4 nodes: `builder.add_node("intake", intakeNode)`, `builder.add_node("compliance", complianceNode)`, `builder.add_node("fraud", fraudNode)`, `builder.add_node("advisor", advisorNode)`
    - Wire edges for Intake -> [Compliance || Fraud] -> Advisor:
      - `builder.add_edge(START, "intake")`
      - `builder.add_edge("intake", "compliance")`
      - `builder.add_edge("intake", "fraud")`
      - `builder.add_edge("compliance", "advisor")`
      - `builder.add_edge("fraud", "advisor")`
      - `builder.add_edge("advisor", END)`
    - This creates the parallel fan-out: after intake, both compliance and fraud run in the same superstep. Advisor waits for both to complete before executing (LangGraph fan-in behavior).
    - Return the builder (uncompiled — compilation happens with checkpointer at runtime)

    Define `async def getCompiledGraph()`:
    - Get settings via getSettings()
    - Create AsyncPostgresSaver using `AsyncPostgresSaver.from_conn_string(settings.postgresDsn)`
    - Call `await checkpointer.setup()` to create checkpointer tables in Postgres
    - Build graph via `buildGraph()`
    - Return `builder.compile(checkpointer=checkpointer)` and the checkpointer (caller needs to manage connection lifecycle)
    - NOTE: The checkpointer connection must be managed as an async context manager. Provide a helper `async def createGraphWithCheckpointer()` that returns the compiled graph and keeps the connection alive. Use `AsyncPostgresSaver.from_conn_string()` as an async context manager.

    Update src/agentic_claims/app.py (modify existing file from Plan 01):

    - In `@cl.on_chat_start`: initialize the graph by calling the async graph factory, store the compiled graph in `cl.user_session`
    - In `@cl.on_message`:
      - Retrieve graph from user_session
      - Create initial ClaimState: `{"claimId": str(uuid.uuid4()), "status": "draft", "messages": [HumanMessage(content=message.content)]}`
      - Invoke graph with `await graph.ainvoke(state, config={"configurable": {"thread_id": threadId}})` where threadId is a unique string per conversation
      - Extract the final messages from the result
      - Send each agent's AIMessage content back to the user as separate Chainlit messages (so user sees output from each agent node)
    - Handle the AsyncPostgresSaver lifecycle properly — create it once on chat start, use it across the session
