Close UAT Gap 4 (searchPolicies → plain-text-question bypass) per Option A of `13-DEBUG-plain-text-question-bypass.md` §5 Recommendation.

Root cause (verbatim from debug doc §3): both Phase 13 defence layers (`preModelHook` directive injection + `postModelHook` drift predicate) are gated on `clarificationPending == True`, but that flag is ONLY raised by `convertCurrency {supported: false}` — never by `searchPolicies` returning policy violations. On the policy-exception turn, no flag is set → no directive injected → validator dormant → LLM emits plain-text questions with zero tool_calls → the entire defence pipeline is bypassed.

Fix strategy (Option A, minimal surface):
1. Extend `searchPolicies` tool return contract with `violationDetected: bool` + `violationSummary: str | None` derived from a post-retrieval numeric-cap scan against claim state.
2. Add a new branch in `postToolFlagSetter.py` mirroring the `convertCurrency` pattern — on `searchPolicies {violationDetected: true}`, set `clarificationPending = True`.

Net effect: the SAME state-flag → preModelHook directive → postModelHook validator pipeline that already works for the convertCurrency/unsupported-currency path now works for the policy-violation path. The LLM is forced to call askHuman (directive + validator rewrite on first drift + escalate on second drift). 13-12 F1 (clear-on-askHuman) becomes non-vacuous here — the user's justification reply arrives as an askHuman ToolMessage, F1 clears the pending flag, Phase 3 submission proceeds.

Important: Do NOT edit `agentSystemPrompt_v5.py` in this plan. Plan 13-15 deletes prompt step 6 in favour of code-enforced cancel detection. Keep F2 in place here so any remaining prompt-path coverage is preserved until 13-15 lands.

Important: Do NOT edit `state.py` in this plan. No new state fields required — we reuse existing `clarificationPending` and `intakeFindings`.

Purpose: Restores the Phase 13 defence-in-depth architecture for the policy-exception turn shape. Without this, the VND-with-violation submission path is defenceless against LLM drift and stalls users in loops.

    ```python
    import re

    _CAP_PATTERNS = [
        # Explicit SGD cap phrasings: "SGD 20", "SGD 20.00", "S$20", "cap of SGD 20"
        re.compile(r"SGD\s*(\d+(?:\.\d+)?)", re.IGNORECASE),
        re.compile(r"S\$\s*(\d+(?:\.\d+)?)", re.IGNORECASE),
    ]


    def _scanForViolation(chunks: list, claimAmountSgd: float | None) -> tuple[bool, str | None]:
        """Scan retrieved chunks for a numeric cap; compare to claim amount.

        Returns (violationDetected, violationSummary).

        Conservative: only returns True when (a) at least one chunk text contains
        an explicit SGD cap pattern AND (b) claimAmountSgd exceeds that cap.
        Any parse failure or missing amount defaults to (False, None).

        Source: 13-DEBUG-plain-text-question-bypass.md §5 Option A — numeric
        cap matching is the minimum viable signal; more sophisticated compliance
        decisions remain LLM-owned.
        """
        if claimAmountSgd is None or not isinstance(chunks, list):
            return False, None
        lowestCapFound: float | None = None
        lowestCapChunk: str | None = None
        for chunk in chunks:
            if not isinstance(chunk, dict):
                continue
            text = chunk.get("text", "")
            if not isinstance(text, str):
                continue
            for pattern in _CAP_PATTERNS:
                for match in pattern.finditer(text):
                    try:
                        cap = float(match.group(1))
                    except (ValueError, IndexError):
                        continue
                    if lowestCapFound is None or cap < lowestCapFound:
                        lowestCapFound = cap
                        lowestCapChunk = text[:160]
        if lowestCapFound is None:
            return False, None
        if claimAmountSgd <= lowestCapFound:
            return False, None
        summary = (
            f"Amount SGD {claimAmountSgd:.2f} exceeds policy cap of "
            f"SGD {lowestCapFound:.2f}. Excerpt: {lowestCapChunk!r}"
        )
        return True, summary
    ```
