<objective>
Restore visibility of substantive agent content (extraction result tables, policy-check summaries, confirmation narratives) that currently disappears into the "Thought for Xs · N tools" thinking panel whenever the LLM packs prose and a tool call into the same AIMessage. Implements **Fix A** from `13-DEBUG-display-regression.md` (H1 confirmed: `on_chat_model_end` routes content+tool_calls into `thinkingEntries` as "reasoning" and never emits `SseEvent.MESSAGE`).

Purpose: Satisfies UAT Gap 1 (pervasive display regression across Tests 1, 4, 6). Per v5 prompt (Phase 1 steps 6–9 and Phase 2 steps 3–5), user-visible prose preceding a tool call is the normal path, not an edge case. The current SSE classifier treats it as reasoning, which breaks the core chat experience.

Output: One modified production file (`sseHelpers.py`) with a narrowly-scoped change inside the existing `on_chat_model_end` handler, plus one new regression test file.
</objective>
