<objective>
Close UAT Gap 6 (tool-name leak in thinking panel) per the root cause diagnosed in `13-DEBUG-tool-name-leak.md`. Two independent leakage sites in `sseHelpers.py`:

1. **STEP_CONTENT leak (line ~1178):** `_summarizeToolOutput` falls through to `f"Completed {toolName}"` for askHuman (no dedicated branch) and any future tool lacking a dedicated branch. Directly renders raw internal name into `#thinkingContent`.
2. **STEP_NAME leak (line ~1114):** `TOOL_LABELS.get(toolName, f"Running {toolName}...")` — currently benign because all 6 tools are mapped, but the fallback is a hygiene risk for any future tool.

Fix strategy (Option C from the debug doc — defensive labels + filter askHuman):
- Add `TOOL_COMPLETION_LABELS` (sibling of existing `TOOL_LABELS`).
- Replace every `f"Completed {toolName}"` in `_summarizeToolOutput` with `TOOL_COMPLETION_LABELS.get(toolName, "Step complete")`.
- Skip STEP_NAME / STEP_CONTENT / `thinkingEntries.append` when `toolName == "askHuman"`.
- Substitute an `askHumanFired` local signal into `_calcProgressPct` to preserve the 66% progress bump.

Important non-changes (confirmed authoritative by debug doc §6):
- `askHuman.py` unchanged — interrupt mechanism works correctly.
- `interruptDetection.py` unchanged — resume contract unaffected.
- Templates (`chat.html`) unchanged — 13-11's `#interruptTarget` palette + cleanup hook stand.
- INTERRUPT event at sseHelpers.py:1507 continues to fire unchanged.

Purpose: Closes the cosmetic but user-reported "askHuman is an internal thing, should not be exposed" issue AND enables 13-11's `#interruptTarget` cleanup hook to be exercised end-to-end (previously the parallel STEP_NAME/STEP_CONTENT leak confused the mental model).

Output: One production file edit (`sseHelpers.py`), one new test file, one test augmentation. Entirely server-side.
</objective>
