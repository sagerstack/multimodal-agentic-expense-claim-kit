# Phase 6.2: Chat UI Refresh + Employee ID Fix - Context

**Gathered:** 2026-04-04
**Status:** Ready for planning

<domain>
## Phase Boundary

Visual overhaul of the Chat Page to match the Stitch "AI Chat Submission (Updated Branding)" design, plus a backend fix for BUG-015 (employee ID extraction). The layout changes are significant: Decision Pathway sidebar replaces current summary panel, bottom horizontal submission table replaces inline summary, message bubbles get new styling, and branding is updated across 3 locations. Multi-receipt support per claim is added (agent asks "upload another?" after each receipt). Employee ID is extracted server-side instead of relying on the LLM.

</domain>

<decisions>
## Implementation Decisions

### Decision Pathway Sidebar (right column)
- Replaces current summary panel as the right column content
- 4 steps mapped 1:1 to SSE tool events: Receipt Uploaded (upload), AI Extraction (extractReceiptFields), Policy Check (searchPolicies), Final Decision (submitClaim)
- Each step shows timestamp, status badge (Completed/In Progress/Pending)
- AI Extraction step shows REAL VLM confidence score and extracted fields summary (merchant, amount, date) from actual tool results
- Timeline RESETS for each new receipt in multi-receipt flow (shows current receipt only)
- Existing thinking panel in chat area is KEPT alongside the sidebar (both coexist)

### Bottom Submission Table
- Horizontal table below the chat area showing all claims
- Columns from DB claims table: claim number, employee ID, merchant, date, amount, currency, receipt thumbnail (clickable to open), status (from DB)
- Session total card on the left with running sum amount and item count
- NO Submit Batch button — each receipt is submitted individually via the agent chat flow
- Rows appear immediately after extractReceiptFields completes (status "Processing"), update to DB status after submission
- Table loads claims from the DATABASE on page load — shows all previously submitted claims, survives browser refresh and session reset
- Current session's in-progress receipts shown alongside historical claims

### Multi-Receipt Flow
- After first receipt is fully processed (extracted, policy-checked, submitted), agent asks: "Would you like to add another receipt?"
- If user says yes, upload area stays active for next receipt
- Each receipt goes through the full pipeline: extract → policy check → confirm → submit
- Each receipt gets its own separate claim ID (simpler, reuses existing submitClaim flow)
- Decision Pathway fully resets (all 4 steps back to Pending) for each new receipt
- Session total card shows running sum across all receipts in the session

### Branding Updates (3 locations)
- Top nav bar: "Expense AI" gradient logo text (from Stitch design)
- Sidebar header: "Luminous Intelligence / Expense Engine Active" text
- Browser tab title: "Expense AI | AI Expense Submission"

### Sidebar Navigation Changes
- Only show "My Claims" nav item (navigates to Chat Page)
- Remove Dashboard, Audit Log, Review links from sidebar (routes still exist, just hidden from nav — needed for Phase 8-9)
- "New Expense" button in sidebar: if active chat history exists, show confirmation popup ("Are you sure?"); on confirm, reset session but keep submission table showing all previous claims from DB

### Message Bubble Styling
- Adopt Stitch design: bot avatar (smart_toy icon), user avatar (person icon), rounded bubbles, timestamps
- "Analysis Complete" badge with intelligence pulse on extraction results
- Quick-reply buttons (confirm/edit) styled to match Stitch

### Employee ID Fix (BUG-015)
- Extract employee ID SERVER-SIDE from user messages, not by the LLM
- Remove EMP-001 example from system prompt entirely
- Support any format: EMP-042, EMP001, plain numeric (1010736), any alphanumeric string
- Inject extracted ID into submitClaim tool call rather than relying on model propagation

### Claude's Discretion
- Exact message bubble border radius and padding values
- Intelligence pulse animation timing
- How to visually distinguish historical claims vs current session items in the bottom table
- Employee ID regex pattern specifics (Claude to determine best extraction approach)
- How to handle the confirmation popup for "New Expense" (Alpine.js modal vs browser confirm)

</decisions>
