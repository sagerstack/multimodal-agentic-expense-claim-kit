<objective>
Rewrite the app.py streaming architecture from buffer-first to progressive rendering with Type A and Type B reasoning capture in the thinking panel.

Purpose: Currently app.py buffers ALL events, discards intermediate LLM text (Type A reasoning), and builds the thinking panel after everything completes. This plan changes the architecture to: (1) capture Type A reasoning (agent text before tool calls) instead of discarding it, (2) capture Type B reasoning tokens (QwQ's reasoning_content field) when available, (3) stream thinking panel content progressively so users see "Thinking..." immediately with entries appearing in real-time, (4) interleave reasoning blocks and tool call summaries chronologically.

Output: Rewritten app.py with progressive streaming and dual-type reasoning capture.
</objective>

    **3. Rewrite _buildThinkingHtml to handle interleaved entries:**

    The function now accepts `thinkingEntries` (list of mixed reasoning/tool entries) instead of just `toolCalls`:

    ```python
    def _buildThinkingHtml(thinkingEntries: list, totalElapsed: float) -> str:
        elapsed = _formatElapsed(totalElapsed)
        toolCount = sum(1 for e in thinkingEntries if e["type"] == "tool")
        toolLabel = "tool" if toolCount == 1 else "tools"

        entryLines = []
        for entry in thinkingEntries:
            if entry["type"] == "reasoning":
                # Type A: agent intermediate text
                text = entry["content"]
                if len(text) > 500:
                    text = text[:500] + "..."
                entryLines.append(
                    f'<div class="thinking-reasoning">{_escapeHtml(text)}</div>'
                )
            elif entry["type"] == "reasoning_b":
                # Type B: model reasoning tokens (QwQ)
                text = entry["content"]
                if len(text) > 500:
                    text = text[:500] + "..."
                entryLines.append(
                    f'<div class="thinking-reasoning thinking-reasoning-b">{_escapeHtml(text)}</div>'
                )
            elif entry["type"] == "tool":
                summary = _summarizeToolOutput(entry["name"], entry["output"])
                tcElapsed = _formatElapsed(entry["elapsed"])
                entryLines.append(
                    f'<div class="thinking-tool">'
                    f'<b>{entry["name"]}</b> '
                    f'<span class="thinking-elapsed">({tcElapsed})</span>'
                    f'<div class="thinking-summary">{summary}</div>'
                    f'</div>'
                )

        entriesHtml = "\n".join(entryLines)
        return (
            f'<details class="thinking-panel">\n'
            f'<summary>Thought for {elapsed} · {toolCount} {toolLabel}</summary>\n'
            f'{entriesHtml}\n'
            f'</details>\n\n'
        )
    ```

    **4. Add HTML escaping utility:**

    Add a helper function to prevent XSS from reasoning text:
    ```python
    import html

    def _escapeHtml(text: str) -> str:
        return html.escape(text)
    ```

    **5. Progressive rendering — "Thinking..." message:**

    After the streaming loop completes, the rendering phase changes:
    - If thinkingEntries has any entries (reasoning or tools): build thinking panel and render
    - The thinking panel HTML is complete at render time since we buffer events (not truly streaming HTML updates — Chainlit doesn't support partial HTML updates on an existing message)
    - However, we create the response message IMMEDIATELY when the first event arrives, showing "Thinking..." and then UPDATE it with the full thinking panel + response when streaming completes

    Implementation:
    - BEFORE the streaming loop: create a placeholder `cl.Message(content="Thinking...")` and send it
    - AFTER the streaming loop: update the message content with thinking panel HTML + final response
    - If no thinkingEntries and direct response: just update with the response text

    This creates the progressive UX: user sees "Thinking..." instantly, then it gets replaced with the full thinking panel and response.

    Specific code change in the render phase:
    ```python
    # Create placeholder immediately
    responseMsg = cl.Message(content="*Thinking...*")
    await responseMsg.send()

    # ... streaming loop ...

    # After streaming, update with full content
    if finalResponse and finalResponse.strip():
        content = ""
        if thinkingEntries:
            content = _buildThinkingHtml(thinkingEntries, totalElapsed)
        content += finalResponse
        responseMsg.content = content
        await responseMsg.update()
    ```

    - Where code passes `toolCalls` to `_buildThinkingHtml`, pass `thinkingEntries`
    - The `_summarizeToolOutput` function stays unchanged

