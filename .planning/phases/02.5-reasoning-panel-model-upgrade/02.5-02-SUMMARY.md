# Phase 2.5 Plan 02: Rewrite Intake Agent System Prompt v2

**One-liner:** Schema-driven intake workflow with mandatory per-value currency conversion, justification/remarks capture, and immediate acknowledgment pattern.

## What This Plan Delivers

Rewrote the intake agent system prompt (v2) to implement:
1. **Schema-driven workflow**: Call getClaimSchema first to discover required/optional fields dynamically
2. **All-values currency conversion**: Convert EVERY monetary value (total AND tax) individually via convertCurrency
3. **Justification capture**: Prompt user for justification when policy violation exists, include in summary and intakeFindings
4. **Remarks capture**: Capture user's upload description as remarks, include in summary and intakeFindings
5. **Immediate acknowledgment**: Start each turn with brief user-facing acknowledgment before tool calls
6. **Enhanced audit trail**: intakeFindings now includes justification and remarks fields for compliance

### Architecture Changes

**System prompt structure:**
- Architecture section now mandates immediate acknowledgment at turn start
- Phase 1 (Extract) now starts with getClaimSchema call, converts all monetary values individually
- Phase 2 (Policy Check) now captures justification if violation, shows justification and remarks rows in summary
- Phase 3 (Submit) now includes justification and remarks in intakeFindings dict
- Tools section adds getClaimSchema and emphasizes per-value convertCurrency calls
- Constraints section mandates schema call first, all-values conversion, audit trail capture

## Technical Implementation

### Changes Made

**File: src/agentic_claims/agents/intake/prompts/agentSystemPrompt.py**

1. **Architecture section update:**
   - Added: "Start each turn with a brief acknowledgment to give immediate feedback"
   - Examples: "Let me process your receipt" or "Checking policies now"
   - Rationale: User sees instant feedback while thinking panel streams

2. **Phase 1 rewrite (Extract and Present):**
   - Step 1: Call getClaimSchema FIRST to discover database schema
   - Step 3: Call convertCurrency for EVERY monetary value individually (total, tax, subtotals)
   - Step 4: Map extracted fields to schema (identify required/optional/missing)
   - Step 6: Capture user description as "remarks"
   - Step 9: Show conversion for EACH converted value separately
   - Removed hardcoded field list, replaced with schema-driven mapping

3. **Phase 2 rewrite (Policy Check):**
   - Added violation path: "You can still submit with a brief justification. Please explain why this expense was necessary."
   - Added Step 4: Wait for user justification, capture text
   - Updated summary table with Justification and Remarks rows
   - Both rows only appear if applicable (conditional display)

4. **Phase 3 rewrite (Submit):**
   - Updated intakeFindings structure to include justification and remarks fields
   - Added IMPORTANT note with example intakeFindings structure
   - Emphasized audit trail purpose

5. **Tools section update:**
   - Added getClaimSchema tool: Returns {claims: [...], receipts: [...]} with field metadata
   - Updated convertCurrency: "Call this for EACH monetary value that needs conversion (total, tax, etc.)"
   - Emphasized "Never compute exchange rates manually"

6. **Constraints section update:**
   - Added: "Always call getClaimSchema before extractReceiptFields on the first turn"
   - Added: "Convert ALL monetary values via convertCurrency — never compute manually"
   - Added: "Capture user justification and remarks in intakeFindings for audit trail"
