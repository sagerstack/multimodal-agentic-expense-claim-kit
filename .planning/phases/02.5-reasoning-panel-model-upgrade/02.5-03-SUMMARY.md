## Decisions Made

**Type A reasoning capture:**
- Before this plan, tokenBuffer was discarded on intermediate generations (hasToolCalls=true)
- Now captured as {"type": "reasoning", "content": tokenBuffer.strip()} in thinkingEntries
- Rationale: Provides visibility into agent's reasoning before each tool call

**Type B reasoning token capture:**
- Check chunk.additional_kwargs.get("reasoning_content") or .get("reasoning")
- Fall back to chunk.response_metadata.get("reasoning_content") or .get("reasoning")
- Accumulate in reasoningBuffer, append to thinkingEntries as {"type": "reasoning_b"}
- Rationale: QwQ via OpenRouter may surface reasoning in either location, langchain_openai mapping uncertain

**Chronological thinkingEntries:**
- Replaced flat toolCalls list with mixed-type thinkingEntries list
- Each entry: {"type": "reasoning"|"tool"|"reasoning_b", "content": ..., [elapsed: ...]}
- Appended in chronological order as events arrive
- Rationale: Preserves temporal relationship between reasoning and tool execution

**Progressive UX:**
- Create placeholder cl.Message("*Thinking...*") immediately before streaming
- Update message.content with full thinking panel + response after streaming completes
- Rationale: Chainlit doesn't support partial HTML updates to existing message, but placeholder provides instant feedback

**HTML escaping:**
- New _escapeHtml(text) utility using html.escape()
- Applied to all reasoning text before inserting into HTML
- Rationale: LLM-generated reasoning could contain HTML/JS injection attempts

**getClaimSchema tool summary:**
- Added to _summarizeToolOutput: "Schema loaded: N claim fields, M receipt fields"
- Rationale: Makes schema discovery visible in thinking panel
