<objective>
Fix three bugs caused by BUG-026 early termination breaking intakeNode post-processing, plus a pre-existing confidence score data gap.

**Root cause**: The BUG-026 fix breaks out of `astream_events` after submitClaim completes, which cancels the intakeNode's post-processing. This means: (1) `claimSubmitted=True` is never checkpointed, so background agents fail the checkpoint guard; (2) `flushSteps()` is never called, so audit_log stays empty; (3) `dbClaimId` is never set in state.

**Bugs fixed:**
- BUG-027: Audit log empty for submitted claims (Decision Pathway all "Awaiting")
- BUG-028: Intake Agent Findings card shows "Extraction confidence data not available"
- BUG-029: Post-submission background agents never execute (checkpoint guard fails)

Output: All three bugs resolved. Background agents run, audit log populated, confidence scores displayed.
</objective>

  <name>Task 1: Fix early termination to let intakeNode complete checkpoint write</name>
  <files>
    src/agentic_claims/web/sseHelpers.py
  </files>
  <action>
    The SSE stream currently breaks out of `astream_events` at line ~714 when `claimSubmittedFlag` is True and `on_chat_model_end` fires. This cancels the intakeNode before it can: (a) set `claimSubmitted=True` in state, (b) call `flushSteps`, (c) set `dbClaimId`.

    **Fix approach**: Instead of breaking out of the `astream_events` loop immediately, set `shouldTerminateEarly = True` but DON'T `break`. Continue consuming events from the generator but suppress SSE output (don't yield tokens/thinking). Let the generator exhaust naturally so the graph completes the intakeNode checkpoint. The intake node still needs to finish its post-processing.

    Specifically, modify the `on_chat_model_end` handler (around line 710-715):
    ```python
    # BUG-026: if submitClaim already completed, we have the final
    # response — stop emitting SSE output but let the graph finish
    # so intakeNode can complete its checkpoint write.
    if claimSubmittedFlag:
        shouldTerminateEarly = True
        # Do NOT break — let astream_events exhaust so the
        # checkpoint saves claimSubmitted=True and dbClaimId
    ```

    Also modify the `on_chat_model_stream` handler to suppress tokens when `shouldTerminateEarly`:
    ```python
    if eventKind == "on_chat_model_stream":
        if pendingToolCalls > 0 or shouldTerminateEarly:
            continue
    ```

    And suppress pathway/table updates when `shouldTerminateEarly` in `on_tool_start` and `on_tool_end`.

    **Critical**: The evaluatorGate will route to `postSubmission` node which fans out to compliance+fraud. We still need to NOT emit those agent events. So also add:
    ```python
    # After the evaluatorGate fires, the graph will enter compliance/fraud
    # nodes. We don't want to stream those — they run via background task.
    # Break AFTER the evaluatorGate event, not after submitClaim.
    ```

    Actually, the better approach: after `shouldTerminateEarly` is set, let the `astream_events` loop continue BUT only until the intakeNode's state is checkpointed. Watch for the evaluatorGate conditional edge event — that's the signal intakeNode completed. Then break.

    Simplest fix: Instead of breaking in the loop, use a flag. After the loop completes (or after we detect the evaluatorGate ran), the background task info will have a valid checkpoint.

    **Implementation detail**: Watch for `event.get("name") == "evaluatorGate"` or `event.get("event") == "on_chain_end"` with the intake node name. When seen after `shouldTerminateEarly`, break out.

    Alternatively (simpler): just remove the `break` statement and let the full graph run through `astream_events`, but suppress all SSE output after `shouldTerminateEarly`. This means compliance/fraud/advisor run inline (not background), but the user doesn't see them. This is simpler but slower (~60-90s). We want to keep the background approach.

    **Recommended fix**: After `shouldTerminateEarly = True`, don't break immediately. Instead, add a secondary break condition: break when we see the next node starting that ISN'T intake (e.g., `on_chain_start` with name `postSubmission` or `compliance` or `fraud`). This means the intakeNode has finished and checkpointed, but we exit before the slow post-submission agents.

    Concretely:
    1. Remove the `break` at line ~715
    2. Add inside the `on_chain_start` or `on_tool_start` handler:
       ```python
       if shouldTerminateEarly and eventKind == "on_chain_start":
           nodeName = event.get("name", "")
           if nodeName in ("postSubmission", "compliance", "fraud", "advisor", "markAiReviewed"):
               break
       ```
    3. Continue suppressing SSE output for all events when `shouldTerminateEarly`.
